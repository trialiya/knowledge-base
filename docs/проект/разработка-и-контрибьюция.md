## 1. Требования к окружению разработчика

| Инструмент | Версия | Назначение |
|---|---|---|
| Java | **25** (toolchain) | Бэкенд |
| Gradle | **9.5.1** (wrapper) | Сборка |
| Node.js | **22.22.3** | Фронтенд + Spotless (Prettier) |
| Yarn | **1.22.22** | Пакетный менеджер |
| PostgreSQL | **17** + pgvector | База данных |

> **Важно:** Node.js и Yarn скачиваются автоматически плагином `com.github.node-gradle.node` для фронтенд-сборки — Gradle сам загрузит Node 22.22.3 и Yarn 1.22.22 в `.gradle/nodejs` и `.gradle/yarn`. **Однако Spotless для фронтенда использует Prettier, которому нужен Node.js локально.** Если Spotless при проверке фронтенда падает с ошибкой — установите Node.js в систему.

## 3. Локальный запуск для разработки

### 3.1. База данных

```bash
cd docker
cp example.env .env
# Отредактируйте .env — укажите AI_API_KEY, AI_EMBED_API_KEY, PROJECT_PATH_MOUNT
docker compose up -d postgres
```

PostgreSQL будет доступен на `localhost:5432`.

### 3.2. Бэкенд

```bash
./gradlew :backend:bootRun
```

Бэкенд запустится на `http://localhost:8080`.

### 3.3. Фронтенд (dev-режим с hot reload)

```bash
./gradlew :frontend:yarnServe
```

Фронтенд запустится на `http://localhost:3000` и будет проксировать API-запросы на `localhost:8080` (настроено в `package.json`: `"proxy": "http://localhost:8080"`).

### 3.4. Полная сборка

```bash
./gradlew build
```

Выполняет: `yarnInstall` → `yarnBuild` → `copyFrontend` (в `backend/build/resources/main/static`) → `bootJar`.

## 5. Форматирование кода (Spotless)

### Бэкенд (Google Java Format)

```bash
./gradlew spotlessApply    # Форматировать
./gradlew spotlessCheck    # Проверить
```

Настройки в `backend/build.gradle`:
- `googleJavaFormat('1.28.0').aosp()` — AOSP-стиль
- `importOrder()` — сортировка импортов
- `removeUnusedImports()` — удаление неиспользуемых импортов
- `endWithNewline()` — завершающий перевод строки

### Фронтенд (Prettier)

```bash
./gradlew :frontend:spotlessApply
```

Настройки в `frontend/build.gradle`:
- `semi: true` — точки с запятой
- `singleQuote: true` — одинарные кавычки
- `trailingComma: 'all'` — запятые везде
- `printWidth: 120` — ширина строки

> ⚠️ **Требуется Node.js локально.** Spotless для фронтенда запускает Prettier через Node.js. В отличие от фронтенд-сборки (где Node скачивается Gradle-плагином автоматически), Spotless использует системный Node.js. Если `spotlessCheck` падает с ошибкой «node: command not found» — установите Node.js 22+ в систему.

## 7. База данных и миграции

Используется **Flyway**. Миграции лежат в `backend/src/main/resources/db/migration/`.

При запуске приложения Flyway автоматически применяет неприменённые миграции.

Настройки в `application.yaml`:
```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    validate-on-migrate: false
```

## 8. Docker-сборка

Dockerfile использует **трёхстадийную сборку**:

1. **Stage 1 (build):** `eclipse-temurin:25-jdk-alpine` + Node.js/Yarn → `./gradlew :backend:bootJar`
2. **Stage 2 (extract):** `eclipse-temurin:25-jre-alpine` → распаковка слоистого JAR (`java -Djarmode=tools -jar app.jar extract --layers`)
3. **Stage 3 (runtime):** `eclipse-temurin:25-jre-jammy` + git + curl → копирование слоёв из extract → запуск через `start.sh`

Entrypoint — `start.sh`:
```sh
#!/bin/sh
exec java $JAVA_OPTS -jar app.jar
```

Сборка образа:
```bash
docker build -t kb-backend -f docker/Dockerfile .
```

Запуск полного стека:
```bash
cd docker
cp example.env .env
docker compose up -d
```

## 10. Virtual Threads

В проекте включены виртуальные потоки Java 25:
```yaml
spring:
  threads:
    virtual:
      enabled: true
```

Это означает, что все HTTP-запросы обрабатываются на виртуальных потоках — не нужно настраивать пулы потоков.

## 11. Tree-sitter (опционально)

Tree-sitter используется для структурного анализа кода в `OutlineService`. Это нативная библиотека, которая подключается через JNI.

**Поддерживаемые языки:** Java, JavaScript, TypeScript, Python.

**Важно:** Tree-sitter — опциональная зависимость. Если нативная библиотека недоступна (например, несовместимая платформа), `OutlineService` автоматически переключается на regex-парсер. Функциональность структурного анализа кода сохраняется, но с меньшей точностью.

**Зависимости в `build.gradle`:**
- `tree-sitter` (core)
- `tree-sitter-java`
- `tree-sitter-javascript`
- `tree-sitter-typescript`
- `tree-sitter-python`

## 13. Dependency Locking

Проект использует Gradle dependency locking. Lock-файлы (`gradle.lockfile`) зафиксированы в репозитории.

Для обновления lock-файлов после изменения зависимостей:
```bash
./gradlew resolveAndLockAll --write-locks
```

> **Важно:** задача `resolveAndLockAll` несовместима с Gradle Configuration Cache и требует флага `--write-locks`. Без него выполнение прервётся с ошибкой.
