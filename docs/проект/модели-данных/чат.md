## Сущности

### ChatMessageEntity
Таблица: `chat_message`. Реализует `Message` (Spring AI) и `Persistable<Long>`.

| Поле | Тип | Описание |
|---|---|---|
| `id` | long | PK. `isNew()` → `id == 0L` |
| `conversationId` | String | FK → chat_topic.conversation_id |
| `content` | String | Текст сообщения |
| `type` | MessageType | USER, ASSISTANT, SYSTEM, TOOL |
| `position` | long | Порядок в беседе |
| `summarized` | boolean | Попало ли в саммари |
| `summary` | boolean | Является ли само сообщение саммари |
| `createdAt` | LocalDateTime | Время создания |
| `meta` | ChatMessageMeta | Метаданные (tool invocations) — сериализуется в JSON через `ChatMessageMetaToJsonConverter` |

Все поля кроме `id` — `final`.

Метод `getMessage()` возвращает Spring AI wrapper в зависимости от type:
- `USER` → `UserChatMessage`
- `ASSISTANT` → `AssistantChatMessage`
- `SYSTEM` → `SystemChatMessage`
- `TOOL` → `ToolChatMessage`

Метод `getInvocations()` → `List<ToolInvocationMeta>` (из `meta`, может быть null).

**SYSTEM-сообщения** используются для хранения tool calls (`saveToolCalls()`) и суммаризаций. При загрузке истории для пользователя (`findChatMessageByConversationId`) SYSTEM-сообщения фильтруются, но их `toolInvocationMetas` прикрепляются к предыдущему AI-сообщению.

### ChatMessageMeta
Record: `List<ToolInvocationMeta> invocations`. Хранится в поле `meta` таблицы `chat_message` как JSON. Конвертер: `ChatMessageMetaToJsonConverter` (Reader/Writer).

### ChatTopicEntity
Таблица: `chat_topic`. Реализует `Persistable<String>` (ID = conversationId).

| Поле | Тип | Описание |
|---|---|---|
| `conversationId` | String | PK, UUID |
| `user` | String | Имя пользователя |
| `isUser` | boolean | `true` = пользователь переименовал вручную, `false` = авто-имя |
| `topic` | String | Название чата |
| `model` | String | ID выбранной AI-модели (`@Nullable`, null = дефолтная) |
| `createdAt` | LocalDateTime | `@CreatedDate` |
| `updatedAt` | LocalDateTime | `@LastModifiedDate` |
| `isNew` | boolean | `@Transient` |

**Приоритет переименования:** если `isUser = true`, TopicFunction не перезаписывает `topic`.

### Spring AI Message Wrappers
В пакете `model.chat.spring`:

| Класс | Назначение |
|---|---|
| `IMessage` | Интерфейс с методами `getContent()`, `getMessageType()`, `getMetadata()` |
| `UserChatMessage` | Оборачивает ChatMessageEntity с type=USER |
| `AssistantChatMessage` | Оборачивает ChatMessageEntity с type=ASSISTANT |
| `SystemChatMessage` | Оборачивает ChatMessageEntity с type=SYSTEM |
| `ToolChatMessage` | Оборачивает ChatMessageEntity с type=TOOL |

### ChatModelProperties
Конфигурационный record (`kb.chat.*` в `application.yaml`).

| Поле | Тип | Описание |
|---|---|---|
| `defaultModel` | ModelOption | Модель по умолчанию |
| `models` | List\<ModelOption\> | Список дополнительных моделей |

`ModelOption` — record: `{ id: String, label: String }`. Метод `isAllowed(id)` проверяет, есть ли модель в списке разрешённых (defaultModel + models).

#### Способы задания моделей

Модели задаются в `application.yaml` в секции `kb.chat`:

```yaml
kb:
  chat:
    default-model:
      id: ${AI_MODEL}              # ID модели (переменная окружения)
      label: ${AI_MODEL_LABEL:Default}  # Отображаемое имя
    models:                        # Дополнительные модели (опционально)
      - id: ${KB_CHAT_MODELS_0_ID}
        label: ${KB_CHAT_MODELS_0_LABEL}
```

**Три уровня выбора модели** (приоритет по убыванию):

1. **Параметр запроса `?model=...`** — модель, явно переданная в `POST /api/chats/{id}/messages` или `POST /api/chats/{id}/messages/stream`. Валидируется через `isAllowed()`.
2. **Сохранённая модель чата** — поле `model` в `chat_topic`. Задаётся через `PUT /api/chats/{id}/model`. Если модель убрали из конфига, она игнорируется (фильтр `isAllowed`).
3. **Дефолтная модель** — `kb.chat.default-model.id`. Используется когда ни параметр запроса, ни сохранённая модель не заданы.

**Связь со Spring AI:** `spring.ai.openai.chat.options.model` ссылается на `${kb.chat.default-model.id}`, поэтому дефолтная модель из `kb.chat` автоматически используется OpenAI-клиентом.

**Валидация:** `ChatModelProperties.isAllowed(id)` разрешает модель если она совпадает с `defaultModel.id` или есть в списке `models`. При передаче неизвестной модели возвращается `400 Bad Request`.

### ToolInvocation
Record в пакете `model.tool`. Используется в рантайме (стриминг, коллектор).

| Поле | Тип | Описание |
|---|---|---|
| `name` | String | Имя инструмента |
| `arguments` | Map\<Object, Object\> | Параметры вызова |
| `status` | ToolInvocationStatus | `STARTED`, `OK`, `ERROR` |
| `error` | String | Сообщение об ошибке (при ERROR) |
| `resultMeta` | Map\<String, ?\> | Метаданные результата (из `ToolCallResultMetaProvider`) — не сериализуется в JSON |
| `resultGist` | String | Краткая сводка результата |

Метод `toMeta()` → `ToolInvocationMeta` (отбрасывает `resultGist`, оставляет `resultMeta`).

### ToolInvocationMeta

| Поле | Тип | Описание |

| `error` | String | Сообщение об ошибке |
| `resultMeta` | Map\<String, ?\> | Метаданные результата |
