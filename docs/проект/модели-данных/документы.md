## Сущности

### DocumentEntity
Таблица: `documents`. Основная сущность — документ или папка.

| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | Первичный ключ |
| `title` | String | Название |
| `type` | String | `"document"` или `"folder"` |
| `parentId` | Long | FK → documents.id (null для корня) |
| `description` | String | Markdown-содержимое (null для папок) |
| `updatedAt` | LocalDateTime | Время последнего изменения |
| `position` | int | Порядок среди siblings (0-based) |
| `isSystem` | boolean | Системные узлы нельзя удалить/переименовать |
| `version` | int | `@Version` — optimistic locking. Инкрементится при любом save() |
| `summary` | String | AI-саммаризация description (null пока не сгенерирована) |
| `summarySourceVersion` | Integer | Значение `descriptionVersion` на момент генерации summary |
| `descriptionVersion` | int | Инкрементится **только** при изменении description. Стартует с 1 |

**Stale-механика:** `isSummaryStale()` → `summarySourceVersion < descriptionVersion`. Rename/move/reorder не делают summary устаревшим.

### DocumentHistoryEntity
Таблица: `document_history`. Иммутабельный снепшот документа **перед** каждым обновлением.

| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `documentId` | Long | FK → documents.id |
| `version` | int | Версия документа на момент снепшота |
| `title` | String | Название на момент снепшота |
| `type` | String | Тип на момент снепшота |
| `description` | String | Содержимое на момент снепшота |
| `updatedAt` | LocalDateTime | Временная метка |
| `summary` | String | AI-summary на момент снепшота |
| `summarySourceVersion` | Integer | |
| `descriptionVersion` | int | |

Строки никогда не мутируются после вставки — нет `@Version`.

### DocumentEmbeddingEntity
Таблица: `document_embeddings`. Один embedding на документ (UNIQUE по documentId).

| Поле | Тип | Описание |
|---|---|---|
| `id` | Long | PK |
| `documentId` | Long | FK → documents.id (UNIQUE) |
| `embedding` | float[] | Вектор, маппится в pgvector через `FloatArrayToVectorConverter` |
| `model` | String | Модель эмбеддинга, например `"text-embedding-3-small"` |
| `updatedAt` | OffsetDateTime | Время последнего обновления |

## DTO

### Document
Лёгкий DTO для ответов create/update/move. `description` = null (для экономии — полный документ через `GET /api/documents/{id}`). Поля summary всегда заполнены.

| Поле | Тип |
|---|---|
| `id` | String |
| `title` | String |
| `type` | String |
| `parentId` | String |
| `description` | String (null) |
| `updatedAt` | LocalDateTime |
| `children` | List\<Document\> |
| `summary` | String |
| `summaryStale` | boolean |
| `summarySourceVersion` | Integer |

### DocumentNode
Полный DTO для `GET /api/documents/{id}` и tree/children. Включает `description`, `children`, `hasChildren`, `system`.

| Поле | Тип |
|---|---|
| `id` | String |
| `title` | String |
| `type` | String |
| `parentId` | String |
| `description` | String |
| `updatedAt` | LocalDateTime |
| `children` | List\<DocumentNode\> |
| `hasChildren` | boolean |
| `system` | boolean |
| `summary` | String |
| `summaryStale` | boolean |
| `summarySourceVersion` | Integer |

### DocumentHistoryEntry
Read-only record для истории изменений.

| Поле | Тип |
|---|---|
| `documentId` | String |
| `version` | int |
| `title` | String |
| `type` | String |
| `description` | String |
| `updatedAt` | LocalDateTime |
| `summary` | String |

### SearchResult
Результат поиска.

| Поле | Тип |
|---|---|
| `id` | String |
| `title` | String |
| `snippet` | String |
| `updatedAt` | LocalDateTime |
| `summary` | String |

### CreateDocumentRequest
| Поле | Тип | По умолчанию |
|---|---|---|
| `title` | String | — |
| `type` | String | `"document"` |
| `parentId` | String | null |
| `description` | String | null |

### UpdateDocumentRequest
| Поле | Тип |
|---|---|
| `title` | String (опционально) |
| `description` | String (опционально) |

### MoveToParentRequest
| Поле | Тип |
|---|---|
| `parentId` | String (null → корень) |

### ReorderRequest
| Поле | Тип |
|---|---|
| `parentId` | String (null → корень) |
| `orderedIds` | List\<String\> (все siblings в нужном порядке) |

### PagedChildren
Record для пагинированных children.

| Поле | Тип |
|---|---|
| `items` | List\<DocumentNode\> |
| `totalElements` | long |
| `totalPages` | int |
| `page` | int |
| `size` | int |
| `hasNext` | boolean |

## Связи

- `documents.parentId` → `documents.id` (само-ссылка, дерево)
- `document_history.documentId` → `documents.id`
- `document_embeddings.documentId` → `documents.id` (UNIQUE)
- `attachments.documentId` → `documents.id` (когда ownerType = 'document')
