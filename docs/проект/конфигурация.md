# Конфигурация

## Основные параметры (`application.yaml`)

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `server.port` | `8080` | Порт backend-сервера |
| `spring.datasource.url` | `jdbc:postgresql://localhost:5432/knowledgebase` | URL подключения к PostgreSQL |
| `spring.datasource.username` | `knowledgebase` | Имя пользователя БД |
| `spring.datasource.password` | `knowledgebase` | Пароль БД |
| `spring.servlet.multipart.max-file-size` | `50MB` | Максимальный размер загружаемого файла |
| `spring.servlet.multipart.max-request-size` | `50MB` | Максимальный размер запроса |
| `spring.mvc.async.request-timeout` | `30m` | Таймаут асинхронных запросов (AI-саммаризация, SSE) |
| `semantic.enabled` | `true` | Включает/отключает семантический поиск и эмбеддинги |
| `kb.documents.export-path` | `${java.io.tmpdir}/kb-doc-export` | Путь для экспорта документов |

### ChatConfig — конфигурация чата

`ChatConfig` (`config/ChatConfig.java`) — центральный класс конфигурации чата. Создаёт бины:

- **ChatMemory** — `MessageWindowChatMemory` с персистентным хранением через `ChatMemoryService`, скользящее окно 50 сообщений
- **ChatClient** — основной клиент Spring AI:
  - **ChatModel** — OpenAI-совместимая модель
  - **System Prompt** — загружается через `@Value("classpath:prompt/sys.md") Resource sysPrompt` и передаётся в `.defaultSystem(sysPrompt)`
  - **MessageChatMemoryAdvisor** — автоматически добавляет историю
  - **ToolCallingManager** — `ToolCallingManager.builder().toolCallbacks(...)` с 5 ToolCallbacks: `TopicFunction`, `MessageLookupFunction`, `DocumentFunction`, `GitFunction`, `AttachmentFunction`

## Профиль H2 (`application-h2.yaml`)

Активируется через `SPRING_PROFILES_ACTIVE=h2`. Использует H2 в PostgreSQL-режиме с файловым хранением.

| Параметр | Значение |
|---|---|
| `spring.datasource.url` | `jdbc:h2:file:./data/kb;MODE=PostgreSQL;DATABASE_TO_LOWER=TRUE` |
| `spring.datasource.driver-class-name` | `org.h2.Driver` |
| `spring.jpa.database-platform` | `org.hibernate.dialect.H2Dialect` |
| `spring.h2.console.enabled` | `true` |
| `spring.h2.console.path` | `/h2-console` |
| `spring.flyway.locations` | `classpath:db/migration-h2` |
| `semantic.enabled` | `false` |

Конфигурация H2 задаётся в классе `H2JdbcConfig` — создаёт `JdbcTemplate` для H2.

## Docker Compose H2 (`docker/docker-compose-h2.yaml`)

```yaml
services:
  backend:
    image: kb-backend-h2
    build:
      context: ..
      dockerfile: docker/Dockerfile
    container_name: kb-backend-h2
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=h2
      - AI_API_KEY=${AI_API_KEY}
      - AI_BASE_URL=${AI_BASE_URL:-https://api.openai.com/}
      - AI_MODEL=${AI_MODEL:-deepseek-v4-flash}
      - AI_EMBED_BASE_URL=${AI_EMBED_BASE_URL:-https://api.openai.com/}
      - AI_EMBED_MODEL=${AI_EMBED_MODEL:-bge-m3}
      - JAVA_OPTS=${JAVA_OPTS:--Xmx200m}
      - PROJECT_PATH_MOUNT=${PROJECT_PATH_MOUNT:-}
      - DOCUMENTS_EXPORT_PATH=${DOCUMENTS_EXPORT_PATH:-../doc-export}
    volumes:
      - kb-h2-data:/app/data
      - ${PROJECT_PATH_MOUNT:-/tmp/kb-project}:/project:ro
      - ${DOCUMENTS_EXPORT_PATH:-../doc-export}:/app/doc-export
    env_file:
      - ./.env
    restart: unless-stopped

volumes:
  kb-h2-data:
```

## Docker Compose (полный стек, `docker/docker-compose.yaml`)

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg17
    container_name: kb-postgres
    environment:
      POSTGRES_DB: knowledgebase
      POSTGRES_USER: knowledgebase
      POSTGRES_PASSWORD: knowledgebase
    ports:
      - "5432:5432"
    volumes:
      - kb-pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U knowledgebase -d knowledgebase"]
      interval: 5s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  backend:
    image: kb-backend
    build:
      context: ..
      dockerfile: docker/Dockerfile
    container_name: kb-backend
    ports:
      - "8080:8080"
    environment:
      - SPRING_DATASOURCE_URL=jdbc:postgresql://postgres:5432/knowledgebase
      - SPRING_DATASOURCE_USERNAME=knowledgebase
      - SPRING_DATASOURCE_PASSWORD=knowledgebase
      - AI_API_KEY=${AI_API_KEY}
      - AI_BASE_URL=${AI_BASE_URL:-https://api.openai.com/}
      - AI_MODEL=${AI_MODEL:-deepseek-v4-flash}
      - AI_EMBED_BASE_URL=${AI_EMBED_BASE_URL:-https://api.openai.com/}
      - AI_EMBED_MODEL=${AI_EMBED_MODEL:-bge-m3}
      - JAVA_OPTS=${JAVA_OPTS:--Xmx200m}
      - PROJECT_PATH_MOUNT=${PROJECT_PATH_MOUNT:-}
      - DOCUMENTS_EXPORT_PATH=${DOCUMENTS_EXPORT_PATH:-../doc-export}
    volumes:
      - ${PROJECT_PATH_MOUNT:-/tmp/kb-project}:/project:ro
      - ${DOCUMENTS_EXPORT_PATH:-../doc-export}:/app/doc-export
    depends_on:
      postgres:
        condition: service_healthy
    env_file:
      - ./.env
    restart: unless-stopped

volumes:
  kb-pgdata:
```

## Dockerfile (трёхстадийная сборка)

Расположен в `docker/Dockerfile`.

| Стадия | Базовый образ | Назначение |
|---|---|---|
| Stage 1 (build) | `eclipse-temurin:25-jdk-noble` | Сборка backend (Gradle) и frontend (npm) |
| Stage 2 (tree-sitter) | `ubuntu:jammy` | Компиляция tree-sitter парсеров (5 языков) |
| Stage 3 (runtime) | `eclipse-temurin:25-jre-noble` | Финальный образ с tree-sitter библиотеками |

Точка входа: `docker/start.sh`.
