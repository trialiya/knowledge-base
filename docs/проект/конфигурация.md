### Модели чата (`kb.chat.*`)

Конфигурация через `ChatModelProperties` (`config/model/ChatModelProperties.java`) — record с префиксом `kb.chat`:

```yaml
kb:
  chat:
    default-model:
      id: ${AI_MODEL}              # ID дефолтной модели
      label: ${AI_MODEL_LABEL:Default}  # Название в UI
    models:                         # Дополнительные модели (опционально)
      - id: ${KB_CHAT_MODELS_0_ID}
        label: ${KB_CHAT_MODELS_0_LABEL}
      - id: ${KB_CHAT_MODELS_1_ID}
        label: ${KB_CHAT_MODELS_1_LABEL}
```

### ChatConfig — конфигурация чата

`ChatConfig` (`config/ChatConfig.java`) — центральный класс конфигурации чата. Создаёт бины:

- **chatRunExecutor** — `ExecutorService` на виртуальных потоках + `DelegatingSecurityContextExecutorService` (переносит `SecurityContext` на worker-поток). `destroyMethod = "shutdown"`
- **ChatMemory** — `MessageWindowChatMemory` с персистентным хранением через `ChatMemoryService`, скользящее окно 50 сообщений
- **SearchAgentService** — только если `kb.search.subagent.enabled=true`. Создаёт sub-agent с ограниченным набором read-only инструментов из `allowed-tools`
- **ChatClient** — основной клиент Spring AI:
  - **ChatModel** — OpenAI-совместимая модель
  - **System Prompt** — загружается через `@Value("classpath:prompt/sys.md") Resource sysPrompt` и передаётся в `.defaultSystem(sysPrompt)`
  - **MessageChatMemoryAdvisor** — автоматически добавляет историю
  - **ToolCallbacks** — `.defaultToolCallbacks(...)` с 6 ToolCallbacks: `TopicFunction`, `MessageLookupFunction`, `DocumentFunction`, `GitFunction`, `AttachmentFunction`, `SearchAgentFunction` (опционально)

### SecurityConfig — HTTP Basic Auth

`SecurityConfig` (`config/SecurityConfig.java`) — stateless HTTP Basic-аутентификация:

- **PasswordEncoder** — `BCryptPasswordEncoder`
- **InMemoryUserDetailsManager** — один пользователь из `SecurityProperties` (`kb.security.username`/`password`)
- **SecurityFilterChain** — CSRF отключён, `SessionCreationPolicy.STATELESS`, `/actuator/health` публичный, всё остальное требует аутентификации

### SecurityProperties

`SecurityProperties` (`config/model/SecurityProperties.java`) — record с префиксом `kb.security`:

```java
@ConfigurationProperties(prefix = "kb.security")
public record SecurityProperties(String username, String password) {
    public SecurityProperties {
        username = (username == null || username.isBlank()) ? "admin" : username;
        password = (password == null || password.isBlank()) ? "admin" : password;
    }
}
```

### sys.md — системный промпт

`prompt/sys.md` — системный промпт ассистента, загружаемый через `@Value` + `Resource`. Определяет:

- **Роль:** «Ассистент для работы с базой знаний»
- **Критические правила:** запрет на выдумку, скрытые инструменты, синтаксис ссылок `[Название](/?doc=ID)`
- **Алгоритм ответа:** вопрос → нужны ли данные → вызов инструмента → ответ
- **Скрытые инструменты:** `recordChatInsights`, `getUserName`, `getCurrentDateTime`, `getOriginalMessages` — вызываются фоново, не упоминаются пользователю
- **Git-репозиторий: общие ограничения** — read-only, только tracked файлы, diff'ы усекаются до 500 строк, файлы >512 КБ возвращаются фрагментом
- **Таблица выбора инструментов** — сопоставление задач и инструментов (вкл. `searchCodebase`)
- **searchFiles vs grepContent** — сравнительная таблица
- **Когда `searchCodebase`, а когда `grepContent`** — правила выбора
- **Цепочки вызовов** — правила оркестрации (поиск → чтение)
- **grepContent: шпаргалка по запросам** — таблица примеров с pattern, pathGlob, regex
- **Ссылки на сообщения** — `[msg:XYZ]` для цитирования позиций в чате

### search-agent.md — промпт sub-agent

`prompt/search-agent.md` — системный промпт search sub-agent, загружаемый через `@Value` + `Resource`. Определяет:

- **Роль:** «специалист по поиску», только чтение
- **Стратегия:** разбить задачу на термины → grep/search → getFileOutline → getFileContent с диапазоном
- **Ограничения git-инструментов**
- **searchFiles vs grepContent** — сравнительная таблица
- **grepContent: шпаргалка по запросам**
- **Жёсткие правила:** не выдумывать, исправлять ошибки аргументов, экономить токены
- **Формат ответа:** Итог, Места (path:line), Связи

## Профиль H2 (`application-h2.yaml`)

Активируется через `SPRING_PROFILES_ACTIVE=h2`. Использует H2 в PostgreSQL-режиме с файловым хранением.

| Параметр | Значение |
|---|---|
| `spring.datasource.url` | `jdbc:h2:file:./data/kb;MODE=PostgreSQL;DATABASE_TO_LOWER=TRUE` |
| `spring.datasource.driver-class-name` | `org.h2.Driver` |
| `spring.jpa.database-platform` | `org.hibernate.dialect.H2Dialect` |
| `spring.h2.console.enabled` | `true` |
| `spring.h2.console.path` | `/h2-console` |
| `spring.flyway.locations` | `classpath:db/migration-h2` |
| `semantic.enabled` | `false` |

Конфигурация H2 задаётся в классе `H2JdbcConfig` — создаёт `JdbcTemplate` для H2.

## Dockerfile (трёхстадийная сборка)

Расположен в `docker/Dockerfile`.

| Стадия | Базовый образ | Назначение |
|---|---|---|
| Stage 1 (build) | `eclipse-temurin:25-jdk-alpine` | Сборка backend (Gradle) и frontend (npm) |
| Stage 2 (extract) | `eclipse-temurin:25-jre-alpine` | Распаковка слоёв JAR для эффективного кэширования |
| Stage 3 (runtime) | `eclipse-temurin:25-jre-jammy` | Финальный образ (git, curl через apt-get, пользователь `kb`) |
