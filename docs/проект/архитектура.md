| Контроллер | Базовый путь | Назначение |

## 10. Ключевые архитектурные решения

1. **Read-only Git** — работа с репозиторием только на чтение через JGit, без возможности изменять файлы
2. **Гибридный поиск** — комбинация keyword (SQL LIKE/tsvector) и semantic (pgvector cosine similarity) с настраиваемыми весами (kw=0.4, sem=0.6)
3. **SSE streaming** — ответы AI передаются через Server-Sent Events для real-time UX
4. **Кэширование эмбеддингов** — LRU-кэш с хешированием текста (SHA-256) для избежания повторных вызовов OpenAI; TTL 30 дней
5. **Системные узлы** — защита от удаления/переименования ключевых документов через флаг `isSystem`
6. **JIRA-чаты** — специальный тип чата с предзагруженным контекстом из JIRA issue и Confluence
7. **Автосуммаризация** — длинные чаты автоматически суммаризируются для экономии контекстного окна
8. **Трёхстадийная Docker-сборка** — разделение build, extract и runtime для минимального финального образа и эффективного кэширования слоёв
9. **Virtual Threads** — включены (`spring.threads.virtual.enabled: true`) для лёгкой конкурентности
10. **История версий документов** — снапшоты перед каждым обновлением, optimistic locking, защита от конкурентных изменений (409 Conflict)
11. **Tree-sitter с фолбэком** — структурный анализ кода через нативную библиотеку tree-sitter, при недоступности — regex-парсер
