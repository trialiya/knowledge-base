## 10. Ключевые архитектурные решения

1. **Read-only Git** — работа с репозиторием только на чтение через нативные git-команды, без возможности изменять файлы
2. **Гибридный поиск** — комбинация keyword (SQL LIKE/tsvector) и semantic (pgvector cosine similarity) с настраиваемыми весами (kw=0.4, sem=0.6)
3. **Фоновые прогоны (Background Runs)** — генерация ответа вынесена из HTTP-запроса в фоновую задачу на виртуальных потоках. Ответ переживает перезагрузку страницы и виден всем вкладкам
4. **SSE event channel** — единый поток событий чата (`GET /api/chats/{id}/events`) обслуживает и стриминг ответа, и кросс-вкладочную синхронизацию. При переподключении реплеятся пропущенные события (seq > fromSeq)
5. **Остановка генерации** — явный сигнал `POST /runs/{runId}/stop` вместо разрыва соединения. Частичный ответ сохраняется с маркером `[stopped]`
6. **Кэширование эмбеддингов** — LRU-кэш с хешированием текста (SHA-256) для избежания повторных вызовов OpenAI; TTL 30 дней
7. **Системные узлы** — защита от удаления/переименования ключевых документов через флаг `isSystem`
8. **Внешние инструменты через MCP** — опциональное подключение внешних MCP-серверов к чат-модели (`kb.mcp.*`), без встроенной интеграции с конкретными сервисами
9. **Автосуммаризация** — длинные чаты автоматически суммаризируются для экономии контекстного окна
10. **Трёхстадийная Docker-сборка** — разделение build, extract и runtime для минимального финального образа и эффективного кэширования слоёв
11. **Virtual Threads** — включены (`spring.threads.virtual.enabled: true`) для лёгкой конкурентности
12. **История версий документов** — снапшоты перед каждым обновлением, optimistic locking, защита от конкурентных изменений (409 Conflict)
13. **Tree-sitter с фолбэком** — структурный анализ кода через нативную библиотеку tree-sitter (опционально). При недоступности нативной библиотеки автоматически используется regex-парсер. Поддерживаются Java, JavaScript, TypeScript, Python
14. **H2 standalone-режим** — возможность запуска без PostgreSQL и без AI-ключа через профиль `h2`. H2 работает в PostgreSQL-совместимом режиме с файловым хранением, семантический поиск отключён (`semantic.enabled: false`)
15. **Интернационализация (i18n)** — полный перевод интерфейса на русский и английский (i18next + react-i18next). Переключатель языка в хедере, автоопределение языка браузера
16. **CI/CD через GitHub Actions** — ежедневная сборка фронта с тестами (только при изменениях), PR-тесты фронта, автообновление зависимостей через Dependabot с перегенерацией lock-файлов. Подробнее — см. **[GitHub Workflows](github-workflows.md)**
17. **HTTP Basic Auth** — stateless-аутентификация через Spring Security. Один пользователь из конфига (`kb.security.username`/`password`), хранение в памяти. `/actuator/health` открыт без аутентификации
18. **SearchAgent (searchCodebase)** — sub-agent многошагового поиска, exposed как инструмент `searchCodebase`. Изолированный цикл grep→read→отчёт с жёстким лимитом итераций. Модель может отличаться от основной (дешевле/быстрее). Защита от рекурсии: `searchCodebase` исключён из allowed-tools сабагента
19. **ChatEventService + ConversationHub** — in-memory pub/sub на каждый чат. Лог событий хранится только для активного прогона (очищается по завершении). Закрытые хабы выгружаются из реестра. Рассчитано на один инстанс; для горизонтального масштабирования добавляется Redis Pub/Sub как релей
