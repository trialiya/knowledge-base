## 9. Docker-сборка

Трёхстадийная сборка в `docker/Dockerfile`. Два варианта Compose: полный стек (PostgreSQL + backend) и облегчённый (только backend с H2). Подробнее — см. **[Конфигурация](конфигурация.md)**.

| Стадия | Базовый образ | Назначение |
|---|---|---|
| **Stage 1 (Build)** | `eclipse-temurin:25-jdk-alpine` | Компиляция backend + frontend (Gradle + Node/Yarn) |
| **Stage 2 (Extract)** | `eclipse-temurin:25-jre-alpine` | Распаковка слоёв JAR для эффективного кэширования |
| **Stage 3 (Runtime)** | `eclipse-temurin:25-jre-jammy` | Запуск JAR (git, curl через apt-get, пользователь `kb`) |

## 10. Ключевые архитектурные решения

1. **Read-only Git** — работа с репозиторием только на чтение через нативные git-команды, без возможности изменять файлы
2. **Гибридный поиск** — комбинация keyword (SQL LIKE/tsvector) и semantic (pgvector cosine similarity) с настраиваемыми весами (kw=0.4, sem=0.6)
3. **SSE streaming** — ответы AI передаются через Server-Sent Events для real-time UX
4. **Кэширование эмбеддингов** — LRU-кэш с хешированием текста (SHA-256) для избежания повторных вызовов OpenAI; TTL 30 дней
5. **Системные узлы** — защита от удаления/переименования ключевых документов через флаг `isSystem`
6. **JIRA-чаты** — специальный тип чата с предзагруженным контекстом из JIRA issue и Confluence
7. **Автосуммаризация** — длинные чаты автоматически суммаризируются для экономии контекстного окна
8. **Трёхстадийная Docker-сборка** — разделение build, extract и runtime для минимального финального образа и эффективного кэширования слоёв
9. **Virtual Threads** — включены (`spring.threads.virtual.enabled: true`) для лёгкой конкурентности
10. **История версий документов** — снапшоты перед каждым обновлением, optimistic locking, защита от конкурентных изменений (409 Conflict)
11. **Tree-sitter с фолбэком** — структурный анализ кода через нативную библиотеку tree-sitter (опционально). При недоступности нативной библиотеки автоматически используется regex-парсер. Поддерживаются Java, JavaScript, TypeScript, Python
12. **H2 standalone-режим** — возможность запуска без PostgreSQL и без AI-ключа через профиль `h2`. H2 работает в PostgreSQL-совместимом режиме с файловым хранением, семантический поиск отключён (`semantic.enabled: false`)
13. **Интернационализация (i18n)** — полный перевод интерфейса на русский и английский (i18next + react-i18next). Переключатель языка в хедере, автоопределение языка браузера
14. **CI/CD через GitHub Actions** — ежедневная сборка фронта с тестами (только при изменениях), PR-тесты фронта, автообновление зависимостей через Dependabot с перегенерацией lock-файлов. Подробнее — см. **[GitHub Workflows](github-workflows.md)**
