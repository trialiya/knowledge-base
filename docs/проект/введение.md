# Введение в Knowledge Base

Knowledge Base — это AI-ассистированная система управления знаниями, которая объединяет структурированную базу знаний, чат с LLM и интеграцию с Git-репозиторием. Проект построен на современном стеке: Java 25, Spring Boot 3.5.15, Spring AI 1.1.6, React 19, PostgreSQL 17 с расширением pgvector.

## Ключевые возможности

1. **База знаний** — древовидная структура документов с drag & drop, вложенные папки, полнотекстовый и семантический поиск
2. **AI-чат** — общение с LLM (OpenAI-совместимые модели), стриминг ответов, контекст из базы знаний
3. **Интеграция с Git** — анализ кода репозитория через AI-инструменты, чтение файлов, коммитов, диффов
4. **Гибридный поиск** — комбинация полнотекстового (pgvector) и семантического (эмбеддинги) поиска
5. **Управление вложениями** — загрузка файлов в документы и чаты, AI-саммаризация вложений
6. **AI-саммаризация** — автоматическая генерация краткого описания документов с отслеживанием актуальности
7. **Экспорт документов** (только API) — выгрузка базы знаний в файловую систему в формате Markdown + YAML, см. [Экспорт документов](../features/экспорт-документов.md)
8. **Jira-интеграция** — создание чатов с контекстом задач Jira
9. **История изменений** — отслеживание версий документов с возможностью просмотра предыдущих состояний
10. **i18n** — интерфейс на русском и английском, переключатель языка

## Быстрый старт

```bash
# Клонирование репозитория
git clone <repository-url>
cd knowledge-base

# Запуск через Docker Compose
docker compose up -d

# Приложение будет доступно на http://localhost:8080
```

### Без Docker — запуск из JAR

```bash
./gradlew :backend:bootJar          # фронтенд встраивается автоматически
SPRING_PROFILES_ACTIVE=h2 \
AI_BASE_URL=https://api.openai.com/ AI_API_KEY=your-key AI_MODEL=gpt-4o \
PROJECT_PATH=./ \
java -jar backend/build/libs/backend-1.0-SNAPSHOT.jar
```

Требуется JDK 25. Подробнее — [Руководство по установке](руководство-по-установке.md), раздел 4.

## Основные характеристики

- **33 эндпоинта** в трёх контроллерах (DocumentController, ChatController, AttachmentController)
- **24 AI-инструмента** в 5 группах (Document, Git, Attachment, Topic, MessageLookup)
- **8 сущностей** — см. [Модели данных](модели-данных/.content.md)
- **Трёхстадийная Docker-сборка** (build → extract → runtime)
- **Tree-sitter** для структурного анализа кода (Java, JavaScript, TypeScript, Python)
- **Virtual Threads** (Project Loom) для конкурентных запросов
