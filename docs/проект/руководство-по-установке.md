# Руководство по установке

> **Связанные документы:** [Введение](введение.md) · [Архитектура](архитектура.md) · [Модели данных](модели-данных/.content.md) · [API Reference](api-reference.md) · [AI-инструменты](ai-инструменты.md) · [Конфигурация](конфигурация.md)

## 1. Требования

| Компонент | Версия | Примечание |
|---|---|---|
| **Java** | 25 (JDK) | Используется в Docker-образе (Eclipse Temurin). При локальном запуске — JDK 25 или совместимая |
| **Spring Boot** | 3.5.14 | Через Gradle plugin `org.springframework.boot` |
| **Spring AI** | 1.1.6 | OpenAI-совместимый API (чат + эмбеддинги) |
| **PostgreSQL** | 17+ | С расширениями `pgvector` и `pg_trgm`. **Не требуется при использовании H2-профиля** |
| **Node.js** | 22+ | Для сборки фронтенда |
| **Yarn** | 1.22+ | Менеджер пакетов для фронтенда |
| **Gradle** | 9.5.1 (wrapper) | Мультипроектная сборка |
| **Docker** | latest | Опционально, для контейнеризации |
| **Tree-sitter** | 0.26.6 | **Опционально.** Нативная библиотека для структурного анализа кода. При недоступности используется regex-фолбэк |

## 2. Быстрый старт (Docker — рекомендуется)

### Вариант A: Полный стек (PostgreSQL + AI)

```bash
cd docker
cp example.env .env
# Отредактируйте .env — укажите AI_BASE_URL, AI_API_KEY, AI_MODEL,
# AI_EMBED_BASE_URL, AI_EMBED_API_KEY, AI_EMBED_MODEL и PROJECT_PATH_MOUNT
docker compose up -d
```

Приложение будет доступно на `http://localhost:8080`.

**Что поднимается:**
- **PostgreSQL 17 + pgvector** — порт `5432:5432`, БД `knowledgebase`
- **Backend (Spring Boot)** — порт `8080:8080`, собирается из Dockerfile (трёхстадийная сборка: build → extract → runtime)

### Вариант B: H2 (без PostgreSQL, без семантического поиска)

```bash
cd docker
cp example.env .env
# Укажите AI_BASE_URL, AI_API_KEY, AI_MODEL и PROJECT_PATH_MOUNT.
# AI_EMBED_* не требуются — семантический поиск отключён.
docker compose -f docker-compose-h2.yaml up -d
```

Приложение будет доступно на `http://localhost:8080`, H2-консоль — `http://localhost:8080/h2-console`.

**Что поднимается:**
- **Backend (Spring Boot)** с профилем `h2` — встроенная H2 в PostgreSQL-режиме, файловое хранение в volume `kb-h2-data`
- **Без PostgreSQL** — не требуется отдельная БД
- **Без эмбеддингов** — семантический поиск отключён, AI-чат работает при наличии `AI_API_KEY`, keyword-поиск и весь остальной функционал работают

> Подробнее — см. **[Быстрый старт с H2](быстрый-старт-с-h2-standalone.md)** и **[Конфигурация](конфигурация.md)**.

## 3. Локальный запуск (для разработки)

### Шаг 1: База данных

**Вариант A — PostgreSQL:**
Убедитесь, что PostgreSQL 17 + pgvector запущен (по умолчанию `localhost:5432`, БД `knowledgebase`).

Инициализация расширений:
```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

**Вариант B — H2 (без PostgreSQL):**
Пропустите этот шаг. При запуске укажите профиль `h2`:
```bash
export SPRING_PROFILES_ACTIVE=h2
```

### Шаг 2: Конфигурация
```bash
cp docker/example.env .env
# Отредактируйте .env под локальное окружение:
# AI_BASE_URL, AI_API_KEY, AI_MODEL — обязательны для AI-чата
# AI_EMBED_BASE_URL, AI_EMBED_API_KEY, AI_EMBED_MODEL — для семантического поиска
# PROJECT_PATH — путь к репозиторию на хосте
# Для H2-профиля AI_EMBED_* не обязательны
```

### Шаг 3: Сборка и запуск backend (фронтенд собирается автоматически)
```bash
./gradlew :backend:bootJar :backend:bootRun
```

### Шаг 4: Разработка с hot-reload фронта
```bash
# Терминал 1: backend
./gradlew :backend:bootRun

# Терминал 2: frontend (отдельный dev-сервер на localhost:3000)
./gradlew :frontend:yarnServe
```

### Шаг 5: Проверка
```bash
curl http://localhost:8080/api/documents/tree
```

## 4. Запуск из собранного JAR

Для прод-подобного запуска без Gradle и Docker соберите исполняемый JAR и
запустите его напрямую через `java -jar`. Фронтенд встраивается в JAR на этапе
сборки (`:frontend:copyFrontend`), поэтому отдельный dev-сервер не нужен.

### Шаг 1: Сборка JAR
```bash
./gradlew :backend:bootJar
```

Готовый артефакт появится в `backend/build/libs/backend-1.0-SNAPSHOT.jar`.

### Шаг 2: Запуск

**Вариант A — H2 (без PostgreSQL):**
```bash
SPRING_PROFILES_ACTIVE=h2 \
AI_BASE_URL=https://api.openai.com/ \
AI_API_KEY=your-key \
AI_MODEL=gpt-4o \
PROJECT_PATH=./ \
java -jar backend/build/libs/backend-1.0-SNAPSHOT.jar
```
Данные H2 сохраняются в каталоге `local-db/` рядом с процессом.

**Вариант B — PostgreSQL (полный стек):**
```bash
AI_BASE_URL=https://api.openai.com/ \
AI_API_KEY=your-key \
AI_MODEL=gpt-4o \
AI_EMBED_BASE_URL=https://api.openai.com/ \
AI_EMBED_API_KEY=your-key \
AI_EMBED_MODEL=text-embedding-3-small \
PROJECT_PATH=./ \
java -jar backend/build/libs/backend-1.0-SNAPSHOT.jar
```
Требуется запущенный PostgreSQL 17 + pgvector (см. раздел 3, Шаг 1, вариант A).

> Требуется установленный JDK/JRE 25. Версию JAR можно переопределить в
> `build.gradle` (`version`). Дополнительные опции JVM передавайте через
> переменную `JAVA_OPTS` или напрямую: `java -Xmx1g -jar ...`.

## 5. Структура Docker-сборки

```
docker/
├── Dockerfile              # Трёхстадийная сборка (build → extract → runtime)
├── docker-compose.yaml     # Compose-файл с postgres + backend
├── docker-compose-h2.yaml  # Compose-файл только backend с H2 (без PostgreSQL)
├── example.env             # Шаблон переменных окружения
├── init.sql                # SQL для инициализации расширений БД
└── start.sh                # Entrypoint для контейнера backend
```

> Подробное описание каждого файла — см. **[Конфигурация](конфигурация.md)**.

## 6. Устранение проблем

| Проблема | Решение |
|---|---|
| **Port already in use** | Проверьте, что порты `5432` и `8080` не заняты другими процессами |
| **AI_API_KEY not set** | Проверьте `.env` — укажите `AI_BASE_URL`, `AI_API_KEY`, `AI_MODEL`. Для H2-профиля AI-чат тоже требует ключ — только `AI_EMBED_*` необязательны |
| **pgvector not found** | Убедитесь, что используется образ `pgvector/pgvector:pg17`. Или переключитесь на H2-профиль |
| **Frontend не собирается** | Проверьте Node.js 22+ и Yarn 1.22+ |
| **Git safe.directory error** | В Dockerfile добавлен `git config --system safe.directory /project` |
| **Docker volume mount** | `PROJECT_PATH_MOUNT` должен быть абсолютным путём на хосте |
| **H2-консоль не открывается** | URL: `http://localhost:8080/h2-console`, JDBC URL: `jdbc:h2:./local-db/h2`, Username: `knowledgebase`, Password: `knowledgebase` |
| **`java -jar` — UnsupportedClassVersionError** | Нужен JDK/JRE 25. Проверьте `java -version` |
| **JAR не найден после сборки** | Артефакт лежит в `backend/build/libs/backend-1.0-SNAPSHOT.jar`; имя зависит от `version` в `build.gradle` |
| **Tree-sitter не загружается** | Это опционально — `OutlineService` автоматически переключится на regex-парсер. Если нужен tree-sitter, убедитесь в совместимости платформы с нативной библиотекой |
