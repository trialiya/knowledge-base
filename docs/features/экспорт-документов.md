# Экспорт документов

Экспорт базы знаний в файловую систему через API `POST /api/documents/admin/export`. Позволяет выгрузить всю структуру документов в читаемом формате — Markdown + опционально YAML-метаданные.

## Формат экспорта

Экспорт создаёт следующую структуру в директории, указанной в `kb.documents.export-path`:

```
export-dir/
├── .index.md              # упорядоченный список всех корневых элементов
├── vvedenie.md             # документ → .md файл (только description, без заголовка)
├── vvedenie.yaml           # метаданные документа (только при meta=true)
├── arkhitektura.md
├── arkhitektura.yaml
├── modeli-dannykh/         # папка (имя: lower-case, спецсимволы → дефисы)
│   ├── .index.md           # упорядоченный список детей папки
│   ├── .content.md         # описание папки (всегда создаётся, может быть пустым)
│   ├── .meta.yaml          # метаданные папки (только при meta=true)
│   ├── dokumenty.md
│   ├── dokumenty.yaml
│   ├── chat.md
│   └── ...
└── features/
    ├── .index.md
    ├── .content.md
    └── ...
```

### Типы файлов

| Файл | Назначение |
|---|---|
| `название.md` | Тело документа — только description, без заголовка (заголовок в метаданных) |
| `название.yaml` | Метаданные документа: id, title, type, parentId, position, updatedAt (только при `meta=true`) |
| `.index.md` | Упорядоченный список детей папки в Markdown; папки ссылаются на `.content.md`, документы — на `.md` |
| `.content.md` | Описание папки (создаётся всегда, даже для пустых папок) |
| `.meta.yaml` | Метаданные папки (только при `meta=true`) |

### Правила именования

- Заголовки документов приводятся к нижнему регистру, спецсимволы и пробелы заменяются на дефисы: `safeName()` → `modeli-dannykh`, `vvedenie`
- Кириллица сохраняется как есть (транслитерация не применяется)
- При совпадении имён добавляется суффикс: `doc.md` → `doc-1.md` → `doc-2.md`
- Служебные файлы папок используют префикс `.` (скрытые)

## Перезапись ссылок

Все внутренние ссылки вида `[Название](/?doc=ID)` в содержимом документов автоматически заменяются на относительные пути файловой системы:

```
[Архитектура](../проект/архитектура.md)  →  [Архитектура](arkhitektura.md)
[Модели данных](../проект/модели-данных/.content.md) →  [Модели данных](modeli-dannykh/.content.md)
```

Это делает экспортированные документы полностью автономными — ссылки работают при просмотре в любом Markdown-редакторе или на GitHub.

## API

### `POST /api/documents/admin/export`

Экспортирует всю базу знаний в файловую систему.

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `meta` | boolean | `false` | Создавать ли `.yaml` и `.meta.yaml` файлы с метаданными |

**Поведение:**

- Если директория экспорта уже существует, существующие файлы перезаписываются
- Экспорт всегда полный — все документы и папки выгружаются заново
- Ответ: `200 OK` с сообщением об успешном экспорте и количеством записанных файлов

**Ошибки:**

| Код | Причина |
|---|---|
| `500` | Ошибка файловой системы (нет прав, нет места и т.д.) |

## Конфигурация

В `application.yaml`:

```yaml
kb:
  documents:
    export-path: ${KB_DOCUMENTS_EXPORT_PATH:./exported-docs}
```

| Параметр | Переменная окружения | Описание |
|---|---|---|
| `kb.documents.export-path` | `KB_DOCUMENTS_EXPORT_PATH` | Путь к директории для экспорта |

## Примечания

- Экспорт доступен только через API — UI на фронтенде отсутствует
- Вложения в экспорт не включаются
- История изменений (document_history) не экспортируется
- Формат оптимизирован для чтения человеком и хранения в Git
