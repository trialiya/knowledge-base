# Чат — руководство пользователя

## 1. Как начать чат

1. Откройте вкладку **Чат** в верхнем меню
2. Нажмите **+ Новый чат** — создастся черновик чата с именем «Новый чат»
3. Введите вопрос в поле ввода и нажмите **Enter**

> 💡 **Черновик:** новый чат создаётся как черновик (без сохранения на сервере). Реальный чат появляется только после отправки первого сообщения — в этот момент генерируется UUID, чат сохраняется в БД, а URL меняется с `/new` на `/<uuid>`. Черновик не виден в списке чатов слева, пока в нём нет сообщений.

> ⚠️ **Защита от потери изменений:** если вы редактируете документ в Базе знаний и переключаетесь на вкладку Чата, появится диалог подтверждения «Несохранённые изменения». Вы можете продолжить редактирование или отменить изменения и перейти в Чат.

### Переключатель языка

В правой части верхней панели (рядом с кнопкой «Обновить») находится переключатель языка интерфейса — кнопка с иконкой глобуса 🌐. Нажмите, чтобы выбрать **RU** (русский) или **EN** (английский). Язык интерфейса меняется мгновенно для всех элементов: чата, базы знаний, подсказок, названий инструментов.

## 2. Выбор AI-модели

Каждый чат может использовать свою AI-модель. Селектор модели (`ModelSelector`) находится в шапке чата **рядом с заголовком** (в одной строке):

- **Выбор модели** — нажмите на название модели, откроется выпадающий список доступных моделей
- **Пометка «(по умолчанию)»** — показывает, какая модель используется по умолчанию
- **Клавиатурная навигация** — стрелки ↑↓, Home, End, Enter для выбора, Escape для закрытия
- **Сохранение выбора** — последняя выбранная модель запоминается в localStorage и применяется к новым чатам (черновикам)
- **Блокировка во время ответа** — во время генерации ответа модель сменить нельзя; если меню было открыто — оно закроется автоматически
- **Для черновика** — модель сохраняется локально (PUT на бэк не шлётся, т.к. чата ещё нет) и уезжает с первым сообщением

## 3. Что можно спросить

Чат подключён к AI-модели и имеет доступ к базе знаний. Можно спрашивать:

| Категория | Примеры |
|---|---|
| Поиск по документам | «Найди документы про архитектуру», «Что написано про Docker?» |
| Содержимое документов | «Покажи содержимое документа Введение» |
| Создание документов | «Создай документ про новую фичу» |
| Обновление документов | «Обнови документ Архитектура — добавь раздел про кэш» |
| Код проекта | «Найди файл docker-compose.yaml», «Покажи содержимое ChatController.java» |
| История коммитов | «Покажи последние 5 коммитов», «Что изменилось в коммите abc123?» |
| Умный поиск | «Как устроена авторизация и где она проверяется?» |

## 4. Tool Calls (инструменты)

Когда AI использует инструменты (поиск, чтение файлов и т.д.), в чате появляются раскрывающиеся блоки:

- **🔄 Выполняется...** — инструмент в процессе работы
- **✅ Успешно** — инструмент отработал
- **❌ Ошибка** — что-то пошло не так

Каждый инструмент имеет свою иконку (например, 🔍 для поиска, 📄 для чтения файлов, 📜 для истории коммитов, 🧭 для умного поиска) и человекочитаемое название на языке интерфейса. Иконки и категории инструментов хранятся в `toolMeta.js`, переводы названий — в i18n-файлах.

### Компактное превью результата

После завершения tool call плашка показывает **компактное превью** результата — краткую сводку того, что вернул инструмент:

- **Поиск документов** — количество найденных документов и их названия
- **Чтение файла** — путь к файлу, размер, количество строк
- **История коммитов** — количество коммитов, автор и сообщение последнего
- **Git grep** — количество совпадений и пример строки
- **Структура файла** — количество символов (классов, методов) и их имена
- **Умный поиск (searchCodebase)** — количество шагов, полнота, длительность

Нажмите на плашку, чтобы увидеть полные детали: название инструмента, переданные параметры и полный результат. При наведении на плашку появляется тултип с компактным превью — он автоматически позиционируется так, чтобы не выходить за границы экрана.

### Связь tool calls с карточками изменений

Инструменты `createDocument` и `updateDocument` возвращают `resultMeta` с полями `id`, `title`, `descriptionVersion`. На основе этих данных под ответом AI формируются карточки изменений (см. раздел 5). Связь обеспечивается через `getDocChangeRef()` в `toolMeta.js`.

## 5. Карточки изменений документов

Когда AI создаёт или обновляет документы через инструменты `createDocument` / `updateDocument`, под ответом появляются **карточки изменений**:

- **📄 Название документа** — имя созданного или изменённого документа (из `resultMeta.title`)
- **Действие** — «Документ создан» или «Документ обновлён»
- **Версия** — номер версии описания (v1, v2, ...; из `resultMeta.descriptionVersion`)
- **«Посмотреть изменения ›»** — кликабельная стрелка

При клике на карточку открывается **история изменений** (HistoryModal) прямо в чате — вы увидите diff между новой и предыдущей версией описания. Зелёным подсвечены добавленные фрагменты, красным с зачёркиванием — удалённые.

Карточки работают и в реальном времени (во время стриминга), и при повторном открытии чата (из истории).

## 6. Ссылки на документы в сообщениях

Когда AI ссылается на документы базы знаний, в сообщениях появляются ссылки вида `[Название](/?doc=ID)`:

- **Наведите курсор** на ссылку — появится превью документа: название, тип, AI-summary (если есть) или сниппет описания, дата обновления
- **Кликните по ссылке** — откроется вкладка «База знаний» с этим документом
- **Кнопка «Открыть →»** в превью — тоже открывает документ в базе знаний

### Якорные ссылки

Ссылки с якорем `#heading` (например, `[Название](/?doc=ID#раздел)`) прокручивают содержимое документа до нужного заголовка внутри превью. Это работает для любых заголовков h1–h6, у которых есть id.

## 7. Копирование сообщений и блоков кода

### Копирование сообщения целиком

В каждом сообщении есть кнопка копирования:

- **AI-сообщения** — кнопка «Копировать» рядом с кнопкой «Source» в шапке сообщения
- **Пользовательские сообщения** — кнопка «Копировать» в тулбаре сообщения (на синем пузыре)

Нажмите — всё содержимое сообщения скопируется в буфер обмена, иконка на мгновение сменится на галочку.

### Блоки кода

Блоки кода в сообщениях отображаются с улучшенным оформлением:

- **Языковой лейбл** — в шапке блока отображается язык (например, `JAVA`, `PYTHON`, `SQL`)
- **Копирование** — при наведении на блок появляется кнопка копирования. Нажмите — код скопируется в буфер обмена, иконка сменится на галочку

## 8. Автоскролл, кнопка «вниз» и подгрузка истории

Чат автоматически прокручивается вниз при появлении новых сообщений:

- **При отправке сообщения** — мгновенный скролл вниз (сбрасывает флаг «отлипания»)
- **Во время стриминга ответа AI** — мгновенная прокрутка к новому тексту (без smooth, чтобы не дёргалось)
- **При ресайзе окна** — если пользователь у низа, чат остаётся у низа (через `ResizeObserver`)
- **Если вы ушли вверх** (читаете историю) — автоскролл приостанавливается, и в правом нижнем углу появляется круглая кнопка **↓** (стрелка вниз)
- **Нажмите кнопку ↓** — чат мгновенно прокрутится к последнему сообщению, кнопка исчезнет

### Подгрузка старых сообщений

Чат не загружает всю историю сразу — только последние 20 сообщений. При прокрутке вверх старые сообщения подгружаются автоматически:

- **Докрутите до верха** — появится индикатор загрузки, и более старые сообщения добавятся в начало чата
- **Позиция скролла сохраняется** — после подгрузки вы останетесь на том же месте (восстановление через `prependRef`: запоминаются `scrollHeight` и `scrollTop` до вставки, после вставки вычисляется новый `scrollTop`)
- **Нет лишних запросов** — если история кончилась, повторные попытки подгрузки не выполняются; параллельные запросы блокируются через `loadingOlderRef`

## 9. Переименование чата

Есть два способа:

- **В списке чатов** — наведите на чат и нажмите кнопку ✎, введите новое имя, нажмите Enter
- **В шапке чата** — кликните по названию чата, отредактируйте, нажмите Enter

После каждого ответа AI также автоматически обновляет название чата, подбирая тему из диалога. **Важно:** если вы переименовали чат вручную, авто-тема больше не будет перезаписывать ваше название — ручное переименование имеет приоритет.

## 10. Список чатов

- **Создать** — кнопка **+ Новый чат**
- **Выбрать** — клик по чату в списке
- **Удалить** — наведите на чат и нажмите иконку корзины (🗑️). Появится диалог подтверждения «Удалить чат» — чат и все его сообщения будут удалены без возможности восстановления

Активный чат сохраняется в URL и localStorage — при обновлении страницы вы вернётесь в тот же чат.

### Черновик

Черновик (новый чат без сообщений) **не показывается** в сайдбаре. При отправке первого сообщения черновик «промоутится» в реальный чат: генерируется UUID, чат сохраняется на сервере, URL меняется с `/new` на `/<uuid>`, и чат появляется в списке.

### Устаревшие ссылки на чаты

Если вы перешли по ссылке на чат, который был удалён, появится сообщение «Чат не найден». Если чат был открыт из памяти браузера (localStorage), а не по прямой ссылке — приложение молча переключится на первый доступный чат.

## 11. Jira-чаты

Можно создать чат, привязанный к задаче Jira:

1. Нажмите **🔗 JIRA чат** в списке чатов
2. Вставьте URL задачи Jira
3. Нажмите **Создать**

Jira-чат показывает в панели вложений описание задачи и комментарии. Кнопка **Обновить** в панели вложений подтягивает свежие данные из Jira.

## 12. Вложения

У каждого чата есть панель вложений:

- **Открыть** — нажмите иконку 📎 в шапке чата
- **Загрузить файл** — кнопка 📎 в панели вложений или в поле ввода сообщения
- **Скачать / Удалить** — кнопки в строке файла

Поддерживаемые форматы: текстовые файлы (.txt, .md, .json, .yaml, .xml, .csv, .log, .sql, .java, .js, .ts, .py, .html, .css и др.)

## 13. Библиотека фраз

В пустом чате (без сообщений) над полем ввода появляется блок **«Готовые фразы»** — набор промптов для быстрого старта. Фразы загружаются из БД через `GET /api/phrases`.

### Категории и фильтры

- **«Все»** — показать все доступные фразы
- **«Избранное»** — только фразы, отмеченные звёздочкой (появляется если есть хотя бы одна)
- **Категории из БД** — динамические кнопки-фильтры (например, «Анализ», «Коммиты»)

### Избранное

У каждой фразы есть кнопка-звезда (⭐). Нажмите — фраза добавится в избранное (оптимистичное обновление UI, `PATCH /api/phrases/{id}/favorite`). Повторное нажатие убирает из избранного. Если снять последнее избранное — фильтр «Избранное» исчезает.

### Использование

- **Клик по чипсу** — вставляет текст фразы в поле ввода (перезапись), курсор в конец
- **Плейсхолдеры** `{{файл}}` и `{{текст}}` — замените на свои значения перед отправкой
- Блок скрывается, как только в чате появляется первое сообщение
- При ошибке загрузки блок просто не показывается (без ошибки в UI)

## 14. Вставка файлов репозитория (`/file`)

Поле ввода чата (`ChipEditor`) поддерживает вставку ссылок на файлы репозитория через триггер `/file`:

### Как использовать

1. Начните вводить `/file ` в поле ввода
2. После пробела начните набирать имя файла — появится выпадающий список (fuzzy-поиск через `GET /api/git/files/search`)
3. Выберите файл стрелками ↑↓ и нажмите Enter — в поле ввода появится **чип** с именем файла

### Чипы файлов

Файл в поле ввода хранится как атомарный токен:

| Токен | Значение |
|---|---|
| `⟦file:PATH⟧` | Весь файл — при отправке раскрывается в fenced code block с содержимым |
| `⟦file:PATH#FROM-TO⟧` | Диапазон строк (1-based включительно) |
| `⟦ref:PATH⟧` | Только ссылка — при отправке раскрывается в `` `PATH` `` |

### Превью содержимого

При клике на чип открывается полноэкранное превью:

- **Режим «с содержимым»** (📄) — показывает код файла с подсветкой (если не бинарный)
- **Режим «только ссылка»** (📎) — показывает только путь к файлу
- **Переключение режима** — кнопка в шапке превью
- **Закрытие** — Escape или клик вне модалки
- **Кеширование** — содержимое файлов кешируется в `contentCache` (Map), повторные запросы не выполняются

### Удаление чипа

Нажмите × на чипе — чип удалится из поля ввода.

## 15. Отправка сообщения и генерация ответа

### Как это работает

Отправка сообщения больше не держит HTTP-запрос открытым до конца генерации. Вместо этого:

1. **Вы нажимаете Enter** — сообщение уходит на сервер через `POST /api/chats/{id}/runs`
2. **Сервер запускает фоновую задачу** (прогон) и сразу возвращает `runId`
3. **Ваше сообщение** появляется в чате оптимистично (сразу, не дожидаясь ответа сервера)
4. **Ответ AI приходит потоком событий** через постоянное SSE-соединение (`GET /api/chats/{id}/events`) — токен за токеном, включая tool calls
5. **По завершении** название чата автоматически обновляется

### Кнопка отправки / остановки

Кнопка в поле ввода меняется в зависимости от состояния:

- **🡅 (стрелка вверх)** — чат свободен, можно отправлять сообщение
- **■ (стоп)** — идёт генерация ответа. Нажмите, чтобы остановить. Частичный ответ сохранится с пометкой «[stopped]»

### Блокировка ввода

Во время генерации ответа поле ввода заблокировано — нельзя отправить новое сообщение, пока текущее не завершится или не будет остановлено.

### Multi-tab синхронизация

Если один и тот же чат открыт в нескольких вкладках:

- **Сообщения из другой вкладки** приходят как события `USER_MESSAGE` — вы видите их в реальном времени
- **Генерация из другой вкладки** тоже видна: появляется кнопка «остановить», ответ стримится
- **Свои сообщения не дублируются** — вкладка-отправитель не показывает эхо своего же сообщения (проверка по `clientMsgId`)

### Resume после перезагрузки

Если вы перезагрузили страницу во время генерации ответа:

- При открытии чата проверяется `GET /api/chats/{id}/runs/active` — есть ли активный прогон
- SSE-соединение подключается с параметром `fromSeq` (последний известный номер события)
- Сервер реплеит все пропущенные события — вы видите уже сгенерированную часть ответа и продолжаете получать новые токены
- Ответ «переживает» перезагрузку страницы

### Конфликт: генерация уже идёт

Если вы попытаетесь отправить сообщение, а в чате уже идёт генерация (возможно, из другой вкладки), сервер вернёт **409 Conflict**. В этом случае:

- Ваше оптимистичное сообщение убирается из чата
- Появляется уведомление «Идёт генерация» с пояснением
- Вы можете дождаться завершения или нажать кнопку «остановить»

## 16. Обработка ошибок

- **Остановка генерации:** если пользователь нажал кнопку Stop, сервер получает явный сигнал `POST /runs/{runId}/stop`. Ответ сохраняется с пометкой `[stopped]`. Событие `RUN_STOPPED` приходит во все вкладки.
- **Ошибка генерации:** если во время фонового прогона произошла ошибка, частичный ответ сохраняется с пометкой `[error]`. Событие `RUN_ERROR` приходит во все вкладки.
- **Обрыв соединения:** SSE-соединение автоматически переподключается с экспоненциальной задержкой (backoff). При переподключении пропущенные события дозагружаются по `fromSeq`.
- **Недоступный чат:** если чат не найден (404) или произошла ошибка загрузки — показывается `ErrorModal` с иконкой и описанием. Поле ввода при этом заблокировано с поясняющим текстом
- **Повторная отправка в недоступный чат:** если чат помечен как `notFound` или `loadError`, при попытке отправить сообщение снова показывается ErrorModal (запрос на сервер не уходит)
