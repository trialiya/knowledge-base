# Чат — руководство пользователя

## 1. Как начать чат

1. Откройте вкладку **Чат** в верхнем меню
2. Нажмите **+ Новый чат** — создастся черновик чата с именем «Новый чат»
3. Введите вопрос в поле ввода и нажмите **Enter**

> 💡 **Черновик:** новый чат создаётся как черновик (без сохранения на сервере). Реальный чат появляется только после отправки первого сообщения. Черновик не виден в списке чатов слева, пока в нём нет сообщений.

> ⚠️ **Защита от потери изменений:** если вы редактируете документ в Базе знаний и переключаетесь на вкладку Чата, появится диалог подтверждения «Несохранённые изменения». Вы можете продолжить редактирование или отменить изменения и перейти в Чат.

### Переключатель языка

В правой части верхней панели (рядом с кнопкой «Обновить») находится переключатель языка интерфейса — кнопка с иконкой глобуса 🌐. Нажмите, чтобы выбрать **RU** (русский) или **EN** (английский). Язык интерфейса меняется мгновенно для всех элементов: чата, базы знаний, подсказок, названий инструментов.

## 2. Выбор AI-модели

Каждый чат может использовать свою AI-модель. Селектор модели находится в шапке чата под заголовком:

- **Выбор модели** — нажмите на название модели, откроется выпадающий список доступных моделей
- **Пометка «(по умолчанию)»** — показывает, какая модель используется по умолчанию
- **Клавиатурная навигация** — стрелки ↑↓, Home, End, Enter для выбора, Escape для закрытия
- **Сохранение выбора** — последняя выбранная модель запоминается и применяется к новым чатам
- **Блокировка во время ответа** — во время генерации ответа модель сменить нельзя

## 3. Что можно спросить

Чат подключён к AI-модели и имеет доступ к базе знаний. Можно спрашивать:

| Категория | Примеры |
|---|---|
| Поиск по документам | «Найди документы про архитектуру», «Что написано про Docker?» |
| Содержимое документов | «Покажи содержимое документа Введение» |
| Создание документов | «Создай документ про новую фичу» |
| Обновление документов | «Обнови документ Архитектура — добавь раздел про кэш» |
| Код проекта | «Найди файл docker-compose.yaml», «Покажи содержимое ChatController.java» |
| История коммитов | «Покажи последние 5 коммитов», «Что изменилось в коммите abc123?» |

## 4. Tool Calls (инструменты)

Когда AI использует инструменты (поиск, чтение файлов и т.д.), в чате появляются раскрывающиеся блоки:

- **🔄 Выполняется...** — инструмент в процессе работы
- **✅ Успешно** — инструмент отработал
- **❌ Ошибка** — что-то пошло не так

Каждый инструмент имеет свою иконку (например, 🔍 для поиска, 📄 для чтения файлов, 📜 для истории коммитов) и человекочитаемое название на языке интерфейса.

### Компактное превью результата

После завершения tool call плашка показывает **компактное превью** результата — краткую сводку того, что вернул инструмент:

- **Поиск документов** — количество найденных документов и их названия
- **Чтение файла** — путь к файлу, размер, количество строк
- **История коммитов** — количество коммитов, автор и сообщение последнего
- **Git grep** — количество совпадений и пример строки
- **Структура файла** — количество символов (классов, методов) и их имена

Нажмите на плашку, чтобы увидеть полные детали: название инструмента, переданные параметры и полный результат. При наведении на плашку появляется тултип с компактным превью — он автоматически позиционируется так, чтобы не выходить за границы экрана.

## 5. Карточки изменений документов

Когда AI создаёт или обновляет документы через инструменты `createDocument` / `updateDocument`, под ответом появляются **карточки изменений**:

- **📄 Название документа** — имя созданного или изменённого документа
- **Действие** — «Документ создан» или «Документ обновлён»
- **Версия** — номер версии описания (v1, v2, ...)
- **«Посмотреть изменения ›»** — кликабельная стрелка

При клике на карточку открывается **история изменений** (HistoryModal) прямо в чате — вы увидите diff между новой и предыдущей версией описания. Зелёным подсвечены добавленные фрагменты, красным с зачёркиванием — удалённые.

Карточки работают и в реальном времени (во время стриминга), и при повторном открытии чата (из истории).

## 6. Ссылки на документы в сообщениях

Когда AI ссылается на документы базы знаний, в сообщениях появляются ссылки вида `[Название](/?doc=ID)`:

- **Наведите курсор** на ссылку — появится превью документа: название, тип, AI-summary (если есть) или сниппет описания, дата обновления
- **Кликните по ссылке** — откроется вкладка «База знаний» с этим документом
- **Кнопка «Открыть →»** в превью — тоже открывает документ в базе знаний

### Якорные ссылки

Ссылки с якорем `#heading` (например, `[Название](/?doc=ID#раздел)`) прокручивают содержимое документа до нужного заголовка внутри превью. Это работает для любых заголовков h1–h6, у которых есть id.

## 7. Копирование сообщений и блоков кода

### Копирование сообщения целиком

В каждом сообщении есть кнопка копирования:

- **AI-сообщения** — кнопка «Копировать» рядом с кнопкой «Source» в шапке сообщения
- **Пользовательские сообщения** — кнопка «Копировать» в тулбаре сообщения (на синем пузыре)

Нажмите — всё содержимое сообщения скопируется в буфер обмена, иконка на мгновение сменится на галочку.

### Блоки кода

Блоки кода в сообщениях отображаются с улучшенным оформлением:

- **Языковой лейбл** — в шапке блока отображается язык (например, `JAVA`, `PYTHON`, `SQL`)
- **Копирование** — при наведении на блок появляется кнопка копирования. Нажмите — код скопируется в буфер обмена, иконка сменится на галочку

## 8. Автоскролл, кнопка «вниз» и подгрузка истории

Чат автоматически прокручивается вниз при появлении новых сообщений:

- **При отправке сообщения** — мгновенный скролл вниз
- **Во время стриминга ответа AI** — плавная прокрутка к новому тексту
- **Если вы ушли вверх** (читаете историю) — автоскролл приостанавливается, и в правом нижнем углу появляется круглая кнопка **↓** (стрелка вниз)
- **Нажмите кнопку ↓** — чат мгновенно прокрутится к последнему сообщению, кнопка исчезнет

### Подгрузка старых сообщений

Чат не загружает всю историю сразу — только последние 20 сообщений. При прокрутке вверх старые сообщения подгружаются автоматически:

- **Докрутите до верха** — появится индикатор загрузки, и более старые сообщения добавятся в начало чата
- **Позиция скролла сохраняется** — после подгрузки вы останетесь на том же месте, новые сообщения появятся выше
- **Нет лишних запросов** — если история кончилась, повторные попытки подгрузки не выполняются

## 9. Переименование чата

Есть два способа:

- **В списке чатов** — наведите на чат и нажмите кнопку ✎, введите новое имя, нажмите Enter
- **В шапке чата** — кликните по названию чата, отредактируйте, нажмите Enter

После каждого ответа AI также автоматически обновляет название чата, подбирая тему из диалога. **Важно:** если вы переименовали чат вручную, авто-тема больше не будет перезаписывать ваше название — ручное переименование имеет приоритет.

## 10. Список чатов

- **Создать** — кнопка **+ Новый чат**
- **Выбрать** — клик по чату в списке
- **Удалить** — наведите на чат и нажмите иконку корзины (🗑️). Появится диалог подтверждения «Удалить чат» — чат и все его сообщения будут удалены без возможности восстановления

Активный чат сохраняется в URL и localStorage — при обновлении страницы вы вернётесь в тот же чат.

### Устаревшие ссылки на чаты

Если вы перешли по ссылке на чат, который был удалён, появится сообщение «Чат не найден». Если чат был открыт из памяти браузера (localStorage), а не по прямой ссылке — приложение молча переключится на первый доступный чат.

## 11. Jira-чаты

Можно создать чат, привязанный к задаче Jira:

1. Нажмите **🔗 JIRA чат** в списке чатов
2. Вставьте URL задачи Jira
3. Нажмите **Создать**

Jira-чат показывает в панели вложений описание задачи и комментарии. Кнопка **Обновить** в панели вложений подтягивает свежие данные из Jira.

## 12. Вложения

У каждого чата есть панель вложений:

- **Открыть** — нажмите иконку 📎 в шапке чата
- **Загрузить файл** — кнопка 📎 в панели вложений или в поле ввода сообщения
- **Скачать / Удалить** — кнопки в строке файла

Поддерживаемые форматы: текстовые файлы (.txt, .md, .json, .yaml, .xml, .csv, .log, .sql, .java, .js, .ts, .py, .html, .css и др.)

## 13. Git-фразы

В пустом чате (без сообщений) над полем ввода появляется блок **«Готовые фразы для работы с Git»** — набор готовых промптов для быстрого старта:

| Категория | Фраза |
|---|---|
| **Анализ** | «История коммитов» — показать историю и объяснить ключевые изменения для файла |
| **Анализ** | «Поиск коммита по тексту» — найти коммит, где была добавлена/удалена строка |
| **Анализ** | «Просмотр изменений» — найти коммит по тексту строки |
| **Коммиты** | «Имя коммита по незакоммиченным» — описать изменения и подготовить имя коммита |

- **Фильтр по категориям** — кнопки «Все», «Анализ», «Коммиты» над списком фраз
- **Клик по чипсу** — вставляет текст фразы в поле ввода, курсор в конец
- **Плейсхолдеры** `{{файл}}` и `{{текст}}` — замените на свои значения перед отправкой
- Блок скрывается, как только в чате появляется первое сообщение

## 14. Обработка ошибок

- **Обрыв соединения:** если во время ответа AI произошёл разрыв соединения (закрыта вкладка, сетевой сбой), накопленный текст ответа сохраняется в чате. При следующем открытии вы увидите частичный ответ
- **Ошибка стрима:** если сервер вернул ошибку во время генерации ответа, частичный ответ сохраняется с пометкой **⚠ Ответ прерван из-за ошибки на сервере**. Если ответ не успел начаться — показывается стандартное сообщение об ошибке
