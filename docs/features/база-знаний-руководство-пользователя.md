# База знаний — руководство пользователя

## 1. Что такое База знаний

База знаний — это древовидное хранилище документов в формате Markdown. Вы можете создавать, редактировать, искать и организовывать документы в папки. Поддерживается гибридный поиск (по ключевым словам + семантический).

Интерфейс базы знаний полностью переведён на русский и английский языки. Переключатель языка 🌐 находится в правой части верхней панели.

## 2. Навигация по дереву

- **Раскрыть/свернуть папку** — клик по стрелке ▶/▼ слева от названия
- **Просмотр документа** — клик по названию узла
- Дерево отображается в левой панели, содержимое — в центральной
- **Drag handle** — значок из шести точек слева от названия узла. Потяните за него, чтобы перетащить документ или папку
- **Системные узлы** — отмечены иконкой 🔒, не могут быть удалены или переименованы

## 3. Просмотр документа

- Документ рендерится как Markdown: заголовки, списки, таблицы, ссылки
- Блоки кода отображаются с языковым лейблом и кнопкой копирования (появляется при наведении)
- **Шапка документа** — показывает breadcrumb (путь от корня), дату создания и вкладки (Summary, Content, Вложения)
- **Вкладка Summary** — содержит две секции:
  - **AI Summary** — краткое содержание от AI (см. раздел 4)
  - **About** — превью содержимого документа (Markdown-рендеринг). Если текст длинный, показываются первые 12 строк с кнопкой «Read more» / «Collapse». В заголовке секции — кнопки: копирования description в буфер, редактирования (переход на вкладку Content), разворачивания на полный экран
- **Якорные ссылки** — ссылки с якорем `#heading` (например, `[Название](/?doc=ID#раздел)`) прокручивают содержимое до нужного заголовка внутри превью документа
- **Полноэкранный режим** — кнопка «Развернуть» (↔) в заголовке секций About и Content. Открывает модальное окно на весь экран для комфортного чтения или редактирования. Закрывается по Esc или клику на фон

## 4. AI-саммаризация

На вкладке **Summary** документа отображается блок «AI Summary» — краткое содержание, сгенерированное искусственным интеллектом.

- **Сгенерировать** — кнопка «Сгенерировать summary» (если summary ещё нет). Если у документа нет описания, кнопка заблокирована с подсказкой «Сначала добавьте описание документа»
- **Актуальность** — если описание документа изменилось после генерации, появляется оранжевый бейдж **«устарел»** и кнопка «Обновить»
- **Перегенерировать** — кнопка доступна всегда, даже если summary актуален
- **Stale-механика** — summary считается устаревшим, только если изменилось содержимое (description). Переименование, перемещение и пересортировка не делают summary устаревшим
- **Индикатор загрузки** — во время генерации кнопка показывает анимированную иконку-спиннер и текст «Генерируется...»

## 5. Поиск

- **Глобальный поиск** — поле поиска находится в верхней части приложения, между вкладками «Чат» и «База знаний» и кнопкой «Обновить». Доступно из любой вкладки
- **Выбор режима** — рядом с полем поиска выпадающий список: Hybrid (по умолчанию), Keyword, Semantic
- **Гибридный поиск**: комбинация keyword + semantic
- **Threshold** — порог релевантности (чем выше, тем строже отбор)
- Результаты подсвечиваются, совпадения выделяются
- **Кнопка «Обновить»** — в глобальной шапке справа от поиска. Принудительно обновляет дерево: перезагружает корень, восстанавливает путь к выбранному документу (с загрузкой всех предков через `materializePathTo`), перезагружает полный документ. Кнопка блокируется на время обновления, показывая анимацию спиннера

## 6. Создание документа

1. Нажмите кнопку **+** над деревом (синяя круглая кнопка со знаком плюс)
2. Выберите родительскую папку (или корень)
3. Введите название
4. Документ создастся и откроется в редакторе

> **Защита от двойного создания:** кнопка «Создать» блокируется на время запроса, чтобы избежать случайного создания дубликатов.

## 7. Редактирование

### Контролируемая модель редактора

MarkdownEditor работает по контролируемой модели:
- **`value`** — текущий текст (черновик), хранится в родителе (`DocumentDetail`/`FolderDetail`)
- **`onChange`** — вызывается при каждом изменении, обновляет черновик
- **`savedValue`** — последнее сохранённое описание
- **Dirty-флаг** — `value !== savedValue`. Когда есть несохранённые изменения, появляется панель с кнопками «Сохранить» / «Отмена»
- **Общий реестр** — `editorDirtyStore` хранит dirty-флаги всех активных редакторов. Используется для защиты навигации (см. раздел «Защита от потери изменений»)
- Благодаря контролируемой модели встроенный и полноэкранный редакторы делят один источник правды — развёрнутое окно открывается с текущими правками

### Тулбар

- **Расширенный тулбар** — кнопки форматирования разбиты на 3 группы с разделителями:
  - **Inline**: заголовок (##), жирный (B), курсив (I), зачёркнутый (S), код (`</>`)
  - **Block**: блок кода, цитата, маркированный список, нумерованный список, чек-лист, горизонтальная линия
  - **Insert**: ссылка, изображение, таблица
- **Правая группа**: кнопка «Копировать всё» (копирует содержимое в буфер, сменяется на галочку на 2 секунды), кнопка Preview (переключает предпросмотр), кнопка «Развернуть» (полноэкранный режим), кнопка «История» (открывает панель истории изменений)

### Горячие клавиши

- **Tab** — вставляет отступ 2 пробела:
  - Без выделения: вставляет 2 пробела в позицию каретки
  - С выделением: добавляет отступ каждой затронутой строке, сохраняя выделение
- **Ctrl+S** — сохранить (или кнопка «Сохранить»)
- **@mention автокомплит** — введите `@` и начните набирать название документа. Появится выпадающий список с результатами поиска. **↑↓** — навигация, **Enter** — вставить ссылку `[Название](/?doc=ID)`, **Esc** — закрыть

### Предпросмотр

- Кнопка «Preview» показывает отрендеренный результат. Блоки кода в предпросмотре отображаются с языковым лейблом и кнопкой копирования
- Поддерживается полный синтаксис Markdown + GitHub Flavored Markdown

### История изменений

Кнопка «История» в тулбаре редактора открывает полноэкранную панель `HistoryModal`:

- **Список версий** — newest-first, по одной записи на каждую уникальную версию описания. Для каждой версии: дата, номер правки, метка «текущая»
- **Ленивая загрузка** — описания версий подгружаются по API только когда версия выбрана, и кэшируются
- **Три режима просмотра:**
  - **Diff** — построчное сравнение двух выбранных версий (через библиотеку `diff`). Добавленные строки (зелёные), удалённые (красные), неизменённые (серые)
  - **Base** — предпросмотр более старой версии (Markdown-рендеринг)
  - **Compare** — предпросмотр более новой версии
- **Overview ruler** — вертикальная полоса прокрутки с цветными маркерами изменений (зелёные/красные). Клик по маркеру — быстрый переход к изменению. Ползунок для перетаскивания
- **Выбор версий** — радиокнопки «База» (старее) и «Изменённая» (новее). Инвариант: база всегда старее изменённой
- **Восстановление** — кнопки «Восстановить базу» и «Восстановить изменённую». Восстановить можно любую версию кроме текущей. После восстановления редактор заполняется текстом выбранной версии
- Подробнее — см. [История описания документа](история-описания-документа.md)

### Защита от потери изменений

- **Диалог подтверждения** — при попытке перейти к другому документу, папке, вкладке или выполнить поиск с несохранёнными изменениями появляется модальное окно «Несохранённые изменения». Вы можете продолжить редактирование или отменить изменения и продолжить навигацию. Механизм реализован через `editorDirtyStore` + `guard()` в `useKnowledgeBase`: действие откладывается в `pendingActionRef`, после подтверждения dirty сбрасывается и действие выполняется
- **Предупреждение браузера** — при закрытии вкладки или обновлении страницы с несохранёнными изменениями браузер покажет стандартное предупреждение (событие `beforeunload`)
- **Сохранение правок при переключении вкладок** — при уходе во вкладку «Чат» с несохранёнными изменениями, редактор не размонтируется (скрывается через CSS), а при возврате документ не перезагружается по API — правки сохраняются

## 8. Удаление

- **Удаление документа** — кнопка удаления в шапке документа. Появляется диалог подтверждения: «Документ будет удалён без возможности восстановления. Продолжить?»
- **Удаление папки** — кнопка удаления в шапке папки. Появляется диалог подтверждения: «Папка и всё её содержимое будут удалены без возможности восстановления. Продолжить?»
- В обоих случаях кнопки: «Удалить» (красная) / «Отмена»

## 9. Drag & Drop

- **Перемещение узлов** — потяните за drag handle (⋮⋮) слева от названия узла
- **Позиции сброса** — при наведении на другой узел подсвечивается позиция: «перед» (верхняя граница), «после» (нижняя граница), «внутрь» (фон узла — только для папок)
- **Подтверждение смены родителя** — если перемещение меняет родительскую папку, появляется модальное окно `MoveConfirmModal`: «Переместить [узел] из [текущая папка] в [новая папка]?»
- **Оптимистичное обновление** — дерево перестраивается мгновенно (функция `applyReorder`), затем отправляется один запрос `PATCH /api/documents/{id}/move` с `parentId` и `afterId`. При ошибке дерево откатывается
- **Ограничения**: нельзя переместить узел в самого себя или в своего потомка; системные узлы не перетаскиваются

## 10. Вложения

- **Просмотр** — вкладка «Вложения» в нижней панели при просмотре документа
- **Загрузка** — кнопка загрузки файла
- **Drag & Drop в редактор** — перетащите файл в Markdown-редактор для вставки ссылки

## 11. Обработка ошибок

- **Документ не найден** — если документ был удалён или URL содержит неверный id, показывается модальное окно с иконкой 🔍 и сообщением «Документ не найден»
- **Ошибка загрузки** — при сетевой ошибке или ошибке сервера показывается модальное окно с иконкой ⚠️ и кодом ошибки (если есть)
- **Ошибка сохранения** — при неудачном сохранении документа показывается модальное окно с сообщением об ошибке. Правки пользователя в UI **не откатываются** — они остаются в редакторе
- **Ошибка перемещения** — при неудачном перемещении дерево откатывается к серверному состоянию (полная перезагрузка корня)

## 12. Экспорт

Экспорт документов доступен только через API (`POST /api/documents/admin/export`). В пользовательском интерфейсе кнопка экспорта отсутствует. Подробнее о структуре экспорта — см. [Экспорт документов](экспорт-документов.md).
