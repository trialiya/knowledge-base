## 3. App.js — корень приложения

**Два таба, оба всегда смонтированы** (скрытие через CSS, не размонтирование). Это сохраняет:
- Позицию скролла
- Загруженные данные
- Состояние стриминга

### Навигация между табами

| Направление | Механизм |
|---|---|
| App → ChatWindow | `onNavigateToDoc(docId)` — переключает таб на Knowledge Base |
| App → KnowledgeBase | `onNavigateToChat(chatId)` — переключает таб на чат |
| ChatWindow → KB | `window.dispatchEvent(new CustomEvent('app:navigate-doc', ...))` |
| KB → ChatWindow | `window.dispatchEvent(new CustomEvent('app:navigate-chat', ...))` |

### URL-схема

```
?view=chat|knowledge&chat=<id>&doc=<id>&tab=<tab>&search=<q>&mode=<m>
```

- `view` — активный таб
- `chat` / `doc` — сосуществуют, каждый компонент читает своё
- `tab` — вкладка внутри Knowledge Base (summary/content/contents/attachments)
- `search` / `mode` — поисковый запрос и режим (hybrid/semantic/keyword)

### ChatWindow — ключевые механики

**Состояние:**
- `chats` — список чатов (id, title, messages, createdAt, jiraUrl, notFound, loadError)
- `activeChatId` — из URL или localStorage
- `isLoading` — флаг стриминга
- `attachPanelOpen` / `attachCount` — панель вложений

**Стриминг SSE:**
1. POST `/api/chat/stream?conversationId=...` с телом сообщения
2. Чтение `response.body.getReader()` — построчная обработка `data:` строк
3. Типы событий:
   - `data: {"message":"текст"}` — текстовый чанк
   - `data: {"toolCall":{...}}` — вызов инструмента
   - `data: {"toolCalls":[...]}` — финальный список вызовов
   - `data: {"finishReason":"TOOL_CALLS"}` — сегмент завершён, начинается новый
   - `data: {"finishReason":"DONE"}` — полное завершение
   - `data: [DONE]` — альтернативный сигнал завершения
4. **Сегментация:** при `TOOL_CALLS` создаётся новое AI-сообщение (если в текущем есть текст)
5. **Abort:** через `AbortController` — кнопка «Стоп»

**Загрузка чатов:**
- `GET /api/chat` → список чатов
- Если `activeChatId` из URL не найден — создаётся заглушка, показывается ошибка
- `failedChatIdsRef` защищает от повторных запросов к несуществующим чатам

**Загрузка сообщений:**
- `GET /api/chat/chat?conversationId=...`
- При 404 — чат помечается `notFound: true`
- При других ошибках — `loadError: status`

**Переименование:**
- `POST /api/chat/topic?conversationId=...` (тело — plain text)
- Фоновое обновление темы после ответа AI: `GET /api/chat/chat?...`

**JIRA-чаты:**
- Создание: `POST /api/chat/jira` → новый чат в списке
- Панель вложений: `JiraAttachmentPanel` вместо `AttachmentPanel`
- Refresh: `POST /api/chat/jira/{id}/refresh`

**Защита от гонки URL:** `ChatWindow` принимает проп `isActive` — синхронизирует URL только когда вкладка чата активна. Это предотвращает перезапись `view=knowledge` при монтировании.

### Message — рендеринг сообщений

**Markdown-рендеринг:**
- `ReactMarkdown` + `remarkGfm`
- Подсветка кода: `SyntaxHighlighter` (Prism, тема vscDarkPlus)
- Переключение «Исходник / Markdown»

**Tool Calls:**
- Группировка последовательных одноимённых вызовов
- Статусы: STARTED (жёлтый пунктир), OK (зелёный круг), ERROR (красный крест)
- Ховер-тултип с деталями (через React Portal)
- Кнопка копирования (имя + аргументы + статус)

**Препроцессинг текста:**
- Добавление пустой строки перед списками для корректного рендеринга Markdown

### MessageInput

- `<textarea>` с авто-resize (max 150px)
- Enter — отправка, Shift+Enter — новая строка
- Кнопка отправки (самолётик) / Стоп (квадрат)
- Кнопка 📎 для быстрой загрузки файла в чат

### ChatList

- Список чатов слева
- Кнопки: «+ Новый чат», «🔗 JIRA чат»
- Inline-редактирование названия (Enter/Escape)
- Удаление (крестик), переименование (карандаш)

### useKnowledgeBase — центральный хук

**Состояние:**
- `tree` — дерево узлов
- `selectedNode` — выбранный узел
- `activeTab` — вкладка (summary/content/contents/attachments)
- `searchQuery`, `searchMode`, `searchResults`
- `showAddModal`, `notFoundDocId`, `docLoadError`, `saveError`
- `refreshing`, `moveConfirm`

**Загрузка дерева:**
- `loadTree()` — корневой уровень через `GET /api/documents/children`
- `handleLoadChildren(parentId, page)` — ленивая загрузка при раскрытии папки
- `refreshScope(parentId)` — после CRUD-операций

**Навигация:**
- `selectNode(node)` — выбор узла. **Всегда** делает `fetchById` для получения полного `description` (бэкенд возвращает сниппет ≤150 символов в tree/children). Для SearchResult (без поля `type`) — также `fetchById` перед отображением.
- `navigateToDocById(docId)` — загрузка цепочки ancestors → раскрытие → выбор
- Восстановление из URL при монтировании
- Обработка `popstate` (браузерный back/forward)

**Защита от дублирующихся запросов:**
- `didInitRef` — предотвращает двойной вызов инициализации в StrictMode
- `inflightChildrenRef` — дедупликация параллельных запросов children (tree expand + folder detail)

**CRUD:**
- Создание: `POST /api/documents`
- Обновление: `PUT /api/documents/{id}` (оптимистичное)
- Удаление: `DELETE /api/documents/{id}` с подтверждением
- Переименование: `handleRename(id, title)`

**Drag & Drop:**
- `handleReorder(dropInfo)` — оптимистичный UI → `PATCH /api/documents/reorder`
- При смене родителя — подтверждение через `MoveConfirmModal`
- Откат при ошибке

**Поиск:**
- `GET /api/search?q=...&mode=...`
- Режимы: hybrid, semantic, keyword

### TreeNode — узел дерева

- **Раскрытие:** `toggleOpenVisual` вместо `toggleOpen` — не делает отдельный fetch, полагается на shared loader из `useFolderChildren`. Это устраняет дублирующиеся запросы при раскрытии папки.
- **«Load more»:** кнопка внизу списка детей, если `totalElements > children.length`
- **Drag & Drop:** module-level `dragState` (обход ограничений браузера)
  - Системные узлы не перетаскиваются
  - Drop-позиции: before, after, inside (для папок)
  - Визуальные индикаторы через CSS-классы
- **Авто-открытие:** `_openOnLoad` для навигации по прямой ссылке
- **Авто-раскрытие:** если выбранный узел — потомок

### useFolderChildren — shared loader

- Проверяет `treeCacheIsComplete` — если дети уже загружены, не делает повторный запрос
- Использует общий `loadChildren` вместо прямого `api.fetchChildren`
- `FolderDetail` передаёт `onLoadChildren` — загрузка детей папки через shared loader, а не отдельный запрос

| Content | MarkdownEditor |

**FolderDetail — 4 вкладки:**

### MarkdownEditor

- **Тулбар:** H2, Bold, Italic, Code, Quote, List, Link
- **Tab:** вставка 2 пробелов
- **Предпросмотр:** переключение редактор/превью
- **Сохранение:** кнопка «Сохранить» + «Отмена» (появляется при изменениях)
- **Preview-only режим:** для SummarySection (без тулбара)

### AttachmentPanel

- **Два режима:** полный (таблица) и компактный (для Summary)
- **Drag & drop:** зона для перетаскивания файлов
- **Загрузка:** `POST /api/{ownerType}/{ownerId}/attachments`
- **Summarize:** `POST /api/attachments/{id}/summarize`
- **Просмотр:** содержимого и описания через `AttachmentModal`
- **Удаление:** с подтверждением через `ConfirmModal`
- **Защита от двойной загрузки:** `loadedOwnerRef` предотвращает повторный fetch при ререндере

### Поиск (SearchResults)

- Карточки результатов: заголовок, дата, хлебные крошки, сниппет
- Хлебные крошки строятся через `findPath()` по дереву

## 7. Потоки данных

### Отправка сообщения в чат
```
MessageInput (Enter)
  → ChatWindow.handleSendMessage(text)
    → setChats (добавляет user + ai сообщения)
    → fetch POST /api/chat/stream (SSE)
      → читаем data: чанки
        → textChanged: обновляем aiMessageTextRef + setChats
        → toolCall: обновляем toolCallsRef + setChats
        → TOOL_CALLS: финализируем сегмент, создаём новый
        → DONE: финализируем всё, fetchAndUpdateTitle
```

### Навигация к документу
```
URL (?doc=123) → useKnowledgeBase.init()
  → fetchAncestors(123) → [rootId, ..., parentId]
  → для каждого: fetchChildren → spliceChildren → _openOnLoad
  → findNodeById → selectNode → fetchById (полный description)
```

### Drag & Drop
```
TreeNode.handleDragStart → dragState.current = payload
TreeNode.handleDragOver → getDropPosition → setDropPos
TreeNode.handleDrop → onReorder(dropInfo)
  → isParentChange? → MoveConfirmModal
  → executeReorder: applyReorder → setTree → PATCH /api/documents/reorder
```
