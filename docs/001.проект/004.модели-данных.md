## 1. Сущности (Entity) — таблицы БД

### 1.1 DocumentEntity — таблица `documents`

Хранит документы и папки базы знаний (иерархическая структура).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `title` | `String` | Название документа/папки |
| `type` | `String` | Тип: `"document"` или `"folder"` |
| `parentId` | `Long` (nullable) | FK → `documents.id`, родительская папка (null = корень) |
| `description` | `String` (nullable) | Содержимое документа (Markdown) |
| `updatedAt` | `LocalDateTime` | Дата последнего изменения |
| `position` | `int` | Порядок отображения среди siblings (zero-based) |
| `isSystem` | `boolean` | Системный узел — нельзя удалить/переименовать (контент редактируем) |
| `version` | `int` | **Оптимистичная блокировка** (`@Version`). Spring Data JDBC автоматически инкрементирует при каждом `save()` и добавляет `AND version = ?` в WHERE. При конфликте → `OptimisticLockingFailureException` → HTTP 409 Conflict |

### 1.2 DocumentHistoryEntity — таблица `document_history`

Иммутабельный снапшот документа, сохраняемый **перед** каждым обновлением. Версия в истории соответствует версии документа **до** инкремента (т.е. для документа с `version=4` в истории будут снапшоты 1, 2, 3).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `documentId` | `Long` (FK) | → `documents.id`, `ON DELETE CASCADE` |
| `version` | `int` | Версия документа на момент снапшота |
| `title` | `String` | Название на момент снапшота |
| `type` | `String` | Тип на момент снапшота |
| `description` | `String` (nullable) | Содержимое на момент снапшота |
| `updatedAt` | `LocalDateTime` | Дата изменения |

Уникальность: `(documentId, version)`. Индекс: `(documentId, version DESC)`.

### 1.3 DocumentEmbeddingEntity — таблица `document_embeddings`

Векторное представление документа для семантического поиска.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `documentId` | `Long` (FK, UNIQUE) | → `documents.id` |
| `embedding` | `float[]` | Вектор (pgvector), конвертация через `FloatArrayToVectorConverter` |
| `model` | `String` | Модель эмбеддинга, e.g. `"text-embedding-3-small"` |
| `updatedAt` | `OffsetDateTime` | Дата обновления |

### 1.4 AttachmentEntity — таблица `attachments`

Файловые вложения, привязанные к документу или чату (полиморфная связь через `ownerType`).

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `ownerType` | `String` | Дискриминатор: `"document"` или `"chat"` |
| `documentId` | `Long` (nullable) | FK → `documents.id` (если ownerType = 'document') |
| `conversationId` | `String` (nullable) | FK → `chat_topic.conversation_id` (если ownerType = 'chat') |
| `fileName` | `String` | Оригинальное имя файла |
| `contentType` | `String` | MIME-тип, e.g. `text/plain`, `text/markdown` |
| `fileSize` | `long` | Размер в байтах |
| `content` | `String` (nullable) | Текстовое содержимое (для текстовых файлов) |
| `summary` | `String` (nullable) | AI-сгенерированное описание |
| `sourceUrl` | `String` (nullable) | URL внешнего источника (JIRA, Confluence) |
| `createdAt` | `OffsetDateTime` | Дата загрузки |
| `updatedAt` | `OffsetDateTime` | Дата последнего изменения |

### 1.5 AttachmentEmbeddingEntity — таблица `attachment_embeddings`

Векторное представление вложения.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `attachmentId` | `Long` (FK, UNIQUE) | → `attachments.id` |
| `embedding` | `float[]` | Вектор (pgvector) |
| `model` | `String` | Модель эмбеддинга |
| `updatedAt` | `OffsetDateTime` | Дата обновления |

### 1.6 ChatMessage — таблица `chat_message`

Сообщение в чате. Реализует Spring AI `Message` и `Persistable<Long>`.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `long` (PK) | Идентификатор |
| `conversationId` | `String` (NOT NULL) | ID беседы |
| `content` | `String` (NOT NULL) | Текст сообщения |
| `type` | `MessageType` | Тип: `TOOL`, `USER`, `SYSTEM`, `ASSISTANT` |
| `position` | `long` | Порядковый номер в беседе |
| `summarized` | `boolean` | Флаг, что сообщение было суммаризировано |
| `summary` | `boolean` | Флаг, что это сообщение — суммаризация |
| `createdAt` | `LocalDateTime` | Дата создания |

Метод `getMessage()` конвертирует entity в Spring AI message:
- `TOOL` → `ToolChatMessage`
- `USER` → `UserChatMessage`
- `SYSTEM` → `SystemChatMessage`
- `ASSISTANT` → `AssistantChatMessage`

### 1.7 ChatTopic — таблица `chat_topic`

Тема/заголовок чат-беседы. Реализует `Persistable<String>`.

| Поле | Тип | Описание |
|---|---|---|
| `conversationId` | `String` (PK) | UUID беседы |
| `user` | `String` | Имя пользователя |
| `isUser` | `boolean` | `true` = пользовательский чат, `false` = системный |
| `topic` | `String` | Тема беседы (3 слова) |
| `createdAt` | `LocalDateTime` | Дата создания |
| `updatedAt` | `LocalDateTime` | Дата последнего обновления |

### 1.8 EmbeddingCacheEntity — таблица `embedding_cache`

Кэш эмбеддингов для избежания повторных вызовов AI API.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `Long` (PK) | Автоинкрементный идентификатор |
| `textHash` | `String` | SHA-256 (lowercase hex, 64 символа) от исходного текста |
| `model` | `String` | Модель эмбеддинга |
| `embedding` | `float[]` | Вектор (pgvector) |
| `createdAt` | `OffsetDateTime` | Дата создания |
| `lastUsedAt` | `OffsetDateTime` | Обновляется при каждом cache hit (для LRU-эвикции) |

Уникальность: `(textHash, model)` — один и тот же текст с разными моделями хранится отдельно.

## 2. DTO (Data Transfer Objects)

### 2.1 Document (doc.dto.Document)

Ответ API для документа (без флага system).

```java
record Document(String id, String title, String type, String parentId,
                String description, LocalDateTime updatedAt, List<Document> children)
```

### 2.2 DocumentNode (doc.dto.DocumentNode)

Узел дерева документов (с флагом system и hasChildren).

```java
record DocumentNode(String id, String title, String type, String parentId,
                    String description, LocalDateTime updatedAt,
                    List<DocumentNode> children, boolean hasChildren, boolean system)
```

> **Важно:** В tree/children-листингах `description` — сниппет ≤150 символов или `null`. Полный текст — только через `GET /api/documents/{id}`.

### 2.3 SearchResult (doc.dto.SearchResult)

Результат поиска по документам.

```java
record SearchResult(String id, String title, String snippet, LocalDateTime updatedAt)
```

### 2.4 CreateDocumentRequest

```java
class CreateDocumentRequest { String title, String type, Long parentId, String description }
```

### 2.5 UpdateDocumentRequest

```java
class UpdateDocumentRequest { String title, String description }
```

### 2.6 ReorderRequest

```java
class ReorderRequest { Long documentId, int newPosition }
```

### 2.7 PagedChildren

Пагинированный список дочерних узлов.

```java
class PagedChildren { List<DocumentNode> children, int totalPages, long totalElements }
```

### 2.8 DocumentHistoryEntry (doc.dto.DocumentHistoryEntry)

Read-only DTO для записи истории изменений.

```java
record DocumentHistoryEntry(String documentId, int version, String title,
                            String type, String description, LocalDateTime updatedAt)
```

### 2.9 Attachment (attachment.dto.Attachment)

Read-only DTO для вложения.

```java
record Attachment(Long id, String ownerType, Long documentId, String conversationId,
                  String fileName, String contentType, long fileSize,
                  String summary, String sourceUrl,
                  OffsetDateTime createdAt, OffsetDateTime updatedAt)
```

### 2.10 Chat (chat.dto.Chat)

DTO чата с сообщениями.

```java
record Chat(String conversationId, String user, String topic,
            LocalDateTime createdAt, LocalDateTime updatedAt, List<ChatMessage> messages)
```

### 2.11 ChatMessage (chat.dto.ChatMessage)

DTO сообщения чата.

```java
record ChatMessage(...) // упрощённая версия
```

### 2.12 StreamMessage

Сообщение для SSE-потока.

```java
record StreamMessage(...)
```

### 2.13 ToolCallMessage / ToolCallsMessage

DTO для вызова инструментов AI.

### 2.14 CreateJiraChatRequest

Запрос на создание JIRA-чата.

```java
class CreateJiraChatRequest { String issueKey, String user }
```

### 2.15 Git DTOs

| DTO | Поля |
|---|---|
| `GitCommit` | `hash`, `shortHash`, `author`, `email`, `date`, `message`, `List<GitDiffEntry> files` |
| `GitDiffEntry` | `status` (A/M/D/R), `path`, `oldPath`, `additions`, `deletions`, `patch` |
| `GitFileContent` | `path`, `content` (null для бинарных), `binary`, `sizeBytes`, `language` (по расширению), `lineCount` (всего строк), `truncated` (true если фрагмент), `fromLine`, `toLine` |
| `GitFileOutline` | `symbols` (List of GitSymbol), `parser` ("tree-sitter" или "regex") |
| `GitSymbol` | `name`, `startLine`, `endLine` |
| `GitFileNode` | `path`, `name`, `type` ("file"/"directory"), `size` |

### 2.16 SemanticSearchResult

Результат векторного поиска.

```java
record SemanticSearchResult(String id, String title, String description,
                            LocalDateTime updatedAt, double similarity)
```

## 4. Связи между таблицами

```
documents (1) ──→ document_embeddings (1)
documents (1) ──→ document_history (*)   [ON DELETE CASCADE]
documents (1) ──→ attachments (*)        [через ownerType='document']
chat_topic (1) ──→ attachments (*)       [через ownerType='chat']
chat_topic (1) ──→ chat_message (*)
attachments (1) ──→ attachment_embeddings (1)
embedding_cache — независимая таблица
```
