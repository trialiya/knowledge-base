## 1. Требования

| Компонент | Версия | Примечание |
|---|---|---|
| **Java** | 25 (JDK) | Eclipse Temurin (jdk-alpine для Docker, jre-alpine для runtime) |
| **Spring Boot** | 3.5.7 | Через Gradle plugin `org.springframework.boot` |
| **Spring AI** | 1.1.6 | OpenAI-совместимый API (чат + эмбеддинги) |
| **PostgreSQL** | 17+ | С расширениями `pgvector` и `pg_trgm` |
| **Node.js** | 22+ | Для сборки фронтенда |
| **Yarn** | 1.22+ | Менеджер пакетов для фронтенда |
| **Gradle** | 8.x (wrapper) | Мультипроектная сборка |
| **Docker** | latest | Опционально, для контейнеризации |

## 2. Быстрый старт (Docker — рекомендуется)

```bash
cd docker
cp example.env .env
# Отредактируйте .env — укажите AI_API_KEY, AI_EMBED_API_KEY и PROJECT_PATH_MOUNT
docker compose up -d
```

Приложение будет доступно на `http://localhost:8080`.

### Что поднимается:
- **PostgreSQL 17 + pgvector** — порт `15432:5432`, БД `knowledgebase`
- **Backend (Spring Boot)** — порт `18080:8080`, собирается из Dockerfile

> Полный список переменных окружения — см. **[Конфигурация](id=10)**.

## 3. Локальный запуск (для разработки)

### Шаг 1: PostgreSQL
Убедитесь, что PostgreSQL 17 + pgvector запущен (по умолчанию `localhost:5432`, БД `knowledgebase`).

Инициализация расширений:
```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

### Шаг 2: Конфигурация
```bash
cp docker/example.env .env
# Отредактируйте .env под локальное окружение
```

### Шаг 3: Сборка и запуск backend (фронтенд собирается автоматически)
```bash
./gradlew :backend:bootJar
./gradlew :backend:bootRun
```

### Шаг 4: Разработка с hot-reload фронта
```bash
# Терминал 1: backend
./gradlew :backend:bootRun

# Терминал 2: frontend (отдельный dev-сервер на localhost:3000)
./gradlew :frontend:yarnServe
```

### Шаг 5: Проверка
```bash
curl http://localhost:8080/api/tree
```

## 4. Структура Docker-сборки

```
docker/
├── Dockerfile          # Многостадийная сборка (build → runtime)
├── docker-compose.yaml # Compose-файл с postgres + backend
├── example.env         # Шаблон переменных окружения
├── init.sql            # SQL для инициализации расширений БД
└── start.sh            # Entrypoint для контейнера backend
```

> Подробное описание каждого файла — см. **[Конфигурация](/?doc=10)**.

## 5. Устранение проблем

| Проблема | Решение |
|---|---|
| **Port already in use** | PostgreSQL на хосте использует `5432` — в Docker проброшен на `15432` |
| **AI_API_KEY not set** | Проверьте `.env` — укажите валидный ключ OpenAI-совместимого API |
| **pgvector not found** | Убедитесь, что используется образ `pgvector/pgvector:pg17` |
| **Frontend не собирается** | Проверьте Node.js 22+ и Yarn 1.22+ |
| **Git safe.directory error** | В Dockerfile добавлен `git config --system safe.directory /project` |
| **Docker volume mount** | `PROJECT_PATH_MOUNT` должен быть абсолютным путём на хосте |
