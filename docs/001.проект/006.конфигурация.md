## Файлы конфигурации

| Файл | Расположение | Назначение |
|---|---|---|
| `application.yaml` | `backend/src/main/resources/application.yaml` | Основная конфигурация |
| `example.env` | `.local/example.env` | Шаблон переменных окружения |
| `docker-compose.yaml` | `docker/docker-compose.yaml` | Docker Compose (PostgreSQL + backend) |

| `init.sql` | `docker/init.sql` | Инициализация расширений БД |
| `start.sh` | `docker/start.sh` | Entrypoint контейнера backend |

## Переменные окружения (`example.env`)

### База данных
| Переменная | Описание | По умолчанию |
|---|---|---|
| `DATASOURCE_URL` | JDBC URL PostgreSQL | `jdbc:postgresql://localhost:5432/knowledgebase` |
| `DATASOURCE_USERNAME` | Пользователь БД | `knowledgebase` |
| `DATASOURCE_PASSWORD` | Пароль БД | `knowledgebase` |

### AI / OpenAI
| Переменная | Описание | По умолчанию |
|---|---|---|
| `AI_API_KEY` | API ключ для чата | — |
| `AI_BASE_URL` | Базовый URL API | — |
| `AI_MODEL` | Модель для чата | — |
| `AI_EMBED_API_KEY` | API ключ для эмбеддингов | `${AI_API_KEY}` |
| `AI_EMBED_BASE_URL` | Базовый URL для эмбеддингов | `${AI_BASE_URL}` |
| `AI_EMBED_MODEL` | Модель эмбеддингов (1024-dim) | — |

### Пути
| Переменная | Описание |
|---|---|
| `PROJECT_PATH` | Путь к корню проекта (монтируется в контейнер) |
| `DOCUMENTS_EXPORT_PATH` | Путь для экспорта документов |

### Atlassian (опционально)
| Переменная | Описание |
|---|---|
| `JIRA_BASE_URL` | URL Jira instance |
| `JIRA_TOKEN` | API токен Jira |
| `CONFLUENCE_BASE_URL` | URL Confluence instance |
| `CONFLUENCE_TOKEN` | API токен Confluence |

## Параметры приложения (`application.yaml`)

### Spring AI — OpenAI
```yaml
spring:
  ai:
    openai:
      base-url: ${AI_BASE_URL}
      api-key: ${AI_API_KEY}
      chat:
        options:
          model: ${AI_MODEL}
          max-tokens: 30000
          temperature: 0.1
          stream-usage: true
          extra-body:
            thinking:
              type: disabled
      embedding:
        base-url: ${AI_EMBED_BASE_URL:${AI_BASE_URL}}
        api-key: ${AI_EMBED_API_KEY:${AI_API_KEY}}
        embeddings-path: /v1/embeddings
        options:
          model: ${kb.embedding.model:bge-m3}
    chat:
      memory:
        repository:
          jdbc:
            initialize-schema: never
      client:
        enabled: off
    retry:
      max-attempts: 2
```

### База данных
```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: ${DATASOURCE_URL:jdbc:postgresql://localhost:5432/knowledgebase}
    username: ${DATASOURCE_USERNAME:knowledgebase}
    password: ${DATASOURCE_PASSWORD:knowledgebase}
  flyway:
    enabled: true
    locations: classpath:db/migration
    validate-on-migrate: false
```

### Эмбеддинги и чанкинг

## Docker Compose (`docker/docker-compose.yaml`)

Два сервиса: **PostgreSQL 17 + pgvector** и **backend** (Spring Boot). MinIO **не используется** — хранение файлов через вложения в PostgreSQL.

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg17
    container_name: kb-postgres
    environment:
      POSTGRES_DB: knowledgebase
      POSTGRES_USER: knowledgebase
      POSTGRES_PASSWORD: knowledgebase
    ports:
      - "15432:5432"
    volumes:
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
      - pgdata:/var/lib/postgresql/data

  backend:
    build:
      context: ..
      dockerfile: docker/Dockerfile
    container_name: kb-backend
    depends_on:
      postgres:
        condition: service_healthy
    env_file:
      - ../.local/example.env
    ports:
      - "18080:8080"
    volumes:
      - ${PROJECT_PATH_MOUNT:-../}:/project:ro
```

## Dockerfile (`docker/Dockerfile`)

Двухстадийная сборка:

1. **Build stage** (`eclipse-temurin:25-jdk-alpine`) — сборка bootJar с кэшированием Gradle/Node/Yarn зависимостей
2. **Runtime stage** (`eclipse-temurin:25-jre-alpine`) — минимальный образ с Git и curl
