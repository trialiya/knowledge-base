# Поиск — руководство пользователя

## 1. Как открыть поиск

Поиск встроен в верхнюю панель раздела «База знаний». Введите запрос в строку поиска и нажмите **Enter** — результаты появятся под строкой.

Горячей клавиши для открытия поиска нет.

## 2. Три режима поиска

Рядом со строкой поиска находится выпадающий список для выбора режима:

| Режим | Как работает | Когда использовать |
|---|---|---|
| **Keyword** | Поиск по точным словам в названии и содержимом | Знаете точные слова из документа |
| **Semantic** | Поиск по смыслу, даже если слова не совпадают | Не помните точных формулировок |
| **Hybrid** | Объединяет keyword и semantic | Универсальный вариант (по умолчанию) |

## 3. Keyword-поиск

Ищет документы по точному совпадению слов в полях `title` и `description`.

**Как работает:**
- Использует SQL `ILIKE` — регистронезависимый поиск
- Ищет каждое слово запроса отдельно
- Ранжирует результаты: совпадение в названии ценится выше, чем в содержимом
- Показывает сниппет — фрагмент текста вокруг найденного слова

**Пример:** запрос `spring ai` найдёт документы, где есть оба слова — «spring» и «ai» (в любом регистре).

## 4. Semantic-поиск

Ищет документы по смысловой близости, даже если точных слов в документе нет.

**Как работает:**
- Текст запроса преобразуется в эмбеддинг (вектор) с помощью модели эмбеддингов
- Сравнивается с эмбеддингами всех документов через **cosine similarity** в pgvector
- Результаты с similarity ниже порога отбрасываются (порог задан на бэкенде, пользователь не меняет)

## 5. Hybrid-поиск

Объединяет keyword и semantic поиск.

**Как работает:**
- Выполняются оба поиска параллельно
- Результаты объединяются с весами (kwWeight и semWeight заданы на бэкенде)
- Дубликаты (один документ, найденный обоими методами) объединяются

Это режим **по умолчанию** — он даёт наилучшие результаты в большинстве случаев.

## 6. Результаты поиска

Каждый результат показывает:

- **Заголовок** — название документа (кликабельно, открывает документ)
- **Breadcrumb** — путь от корня до документа (например: `Проект > Архитектура`)
- **Сниппет** — фрагмент содержимого с выделенными совпадениями
- **Дата обновления** — когда документ был последний раз изменён

**Как работает открытие:** при клике на результат поиска фронтенд запрашивает полный документ через `GET /api/documents/{id}` (поскольку SearchResult содержит только id, title и snippet), и только после получения полных данных открывает документ в панели просмотра.

## 7. Советы

- **Не помните точных слов?** Используйте **semantic** режим
- **Знаете название документа?** Используйте **keyword** режим
- **В большинстве случаев** оставьте **hybrid** — он даёт лучшее из обоих миров
