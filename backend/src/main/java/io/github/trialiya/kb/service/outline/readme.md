## Подводные камни tree-sitter

- **Платформозависимость.** Нативная библиотека грузится под конкретную ОС/архитектуру.
  Сборочный и рантайм-образ должны совпадать (linux-x64, mac-arm64 и т.д.), иначе
  `UnsatisfiedLinkError`. Код это переживает: конструктор `TreeSitterOutlineParser` ловит
  `Throwable`, ставит `available=false`, и `OutlineService` уходит в regex-фолбэк.
- **Потокобезопасность.** `TSParser` не потокобезопасен — создаётся свежий на каждый вызов.
  `TSLanguage` иммутабелен и кэшируется.
- **Если tree-sitter не нужен.** Можно вообще не добавлять зависимости — тогда конструктор
  упадёт в фолбэк, и outline будет работать через regex. Это валидный режим.
